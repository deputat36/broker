# Чек-лист аналитики

## Что уже подготовлено

В проекте есть условное подключение Яндекс Метрики через `_config.yml`.

Пока значение пустое:

```yaml
yandex_metrika_id: ""
```

код счетчика не выводится.

События отправляются функцией `sendGoal()` из файла:

```text
assets/js/main.js
```

Правила рекламных ссылок и QR-кодов описаны в:

```text
docs/utm-guidelines.md
```

## Как включить Метрику

1. Создать отдельный счетчик для домена `sterlikova-ipoteka.ru`.
2. Взять числовой ID счетчика.
3. Вставить ID в `_config.yml`.
4. Не менять остальную структуру кода счетчика без отдельной причины.
5. Дождаться успешной сборки и деплоя GitHub Pages.
6. Проверить главную и несколько внутренних страниц.

Пример:

```yaml
yandex_metrika_id: "12345678"
```

После подключения в HTML должен появиться объект:

```javascript
window.siteAnalytics = { yandexMetrikaId: "12345678" };
```

Фактический тип значения зависит от вывода Jekyll, но ID не должен быть пустым.

## Цели, которые нужно создать

В Метрике нужно создать JavaScript-цели с точными именами:

- `phone_click` — нажатие на ссылку `tel:`;
- `vk_click` — переход на страницу Татьяны ВКонтакте;
- `max_copy` — копирование номера для MAX;
- `consultation_click` — переход на страницу консультации;
- `contacts_click` — переход на страницу контактов;
- `calculator_input` — первое изменение полей ипотечного калькулятора.

Имена целей чувствительны к написанию. Не добавлять пробелы, заглавные буквы и другие варианты одного события.

Цель `lead_submit` пока не создавать как рабочую цель сайта: публичная форма заявок еще не включена.

## Проверка загрузки счетчика

После деплоя открыть сайт и проверить в консоли браузера:

```javascript
typeof window.ym
```

Ожидаемый результат после загрузки счетчика:

```text
function
```

Проверить ID:

```javascript
window.siteAnalytics
```

Объект должен содержать непустой `yandexMetrikaId`.

Ошибки аналитики не должны ломать меню, копирование телефона и калькулятор. Это предусмотрено в `sendGoal()` через безопасную обработку исключений.

## Проверка целей после подключения

Проверять каждое событие отдельно:

1. Открыть сайт в новой вкладке.
2. Выполнить только одно целевое действие.
3. Убедиться, что само действие работает для пользователя.
4. Проверить поступление события в режиме отладки или отчете Метрики.
5. Повторить на мобильном устройстве.

Проверить:

- клик по телефону в шапке;
- клик по телефону в CTA-блоке;
- клик по телефону в мобильной нижней панели;
- переход во ВКонтакте;
- копирование номера для MAX;
- переход на консультацию из меню и контента;
- переход на контакты;
- первое изменение данных в ипотечном калькуляторе.

`calculator_input` должен отправляться один раз за одно открытие формы, а не при каждом нажатии клавиши.

## Проверка UTM-разметки

Для теста использовать отдельную ссылку, например:

```text
https://sterlikova-ipoteka.ru/konsultaciya/?utm_source=test&utm_medium=manual&utm_campaign=analytics_launch&utm_content=check_01
```

Проверить:

- страница открывается без ошибки;
- UTM-параметры остаются в адресной строке после загрузки;
- переход появляется в аналитике с указанными значениями;
- выполненная цель связывается с тестовой кампанией;
- основной домен не перенаправляет пользователя на технический GitHub Pages URL;
- HTTPS не удаляет параметры ссылки.

После проверки тестовые кампании не использовать в реальной рекламе.

## Минимальный набор отчетов после запуска

После накопления данных отслеживать:

- источники переходов;
- рекламные кампании;
- целевые страницы входа;
- количество кликов по телефону;
- количество переходов во ВКонтакте;
- количество копирований телефона для MAX;
- переходы на консультацию;
- использование калькулятора;
- конверсию по отдельным UTM-кампаниям.

Главная задача аналитики — не количество просмотров само по себе, а понимание, какие страницы и рекламные источники приводят к целевым действиям.

## Приемка перед рекламой

Аналитика готова к рекламному трафику, если:

- ID счетчика указан в `_config.yml`;
- workflow завершился успешно;
- счетчик загружается на основном домене;
- все шесть текущих целей созданы с точными именами;
- каждое событие проверено вручную;
- UTM-ссылка корректно определяется;
- телефон, MAX, ВКонтакте и калькулятор продолжают работать при блокировке счетчика;
- тест проведен на компьютере и телефоне.

## Важно

Без согласования не менять:

- ID счетчика;
- телефон;
- ссылку ВКонтакте;
- список публичных каналов связи;
- логику обработки персональных данных;
- запуск цели `lead_submit` и публичной формы.

Не передавать через UTM-параметры имя, телефон, адрес объекта, доход или другие персональные данные клиента.
