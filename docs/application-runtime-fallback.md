# Runtime-резерв онлайн-заявки

## Зачем нужен резерв

Онлайн-анкета зависит от JavaScript: он проверяет поля, формирует безопасный payload, готовит текст и отправляет обращение через настроенный канал.

Если JavaScript полностью отключён либо основной контроллер не загрузился, кнопка формы не должна выглядеть рабочей и молча ничего не делать.

## Поведение страницы

Основная кнопка находится в состоянии `disabled` и `aria-busy="true"` прямо в HTML.

Только после полной инициализации `online-application.js`:

- форма получает `data-application-ready="true"`;
- кнопка включается;
- снимается `aria-busy`;
- начальное сообщение о загрузке очищается;
- runtime-резерв остаётся скрытым.

Если контроллер не завершил инициализацию за 5 секунд, watchdog:

- показывает резервный блок;
- сообщает, что форма не загрузилась;
- предлагает телефон, SMS и ВКонтакте;
- не пытается отправлять введённые сведения другим способом.

При полностью отключённом JavaScript тот же набор контактов выводится через `<noscript>`.

## Проверка

```bash
python3 scripts/audit-application-runtime-fallback.py ./_site
```

Аудит проверяет:

- заблокированную до инициализации кнопку;
- маркер успешной готовности;
- включение кнопки только контроллером;
- watchdog частичного сбоя;
- `<noscript>`-резерв;
- телефон, SMS и ВКонтакте;
- отсутствие runtime-блока на других страницах.

Проверка входит в обязательный `post-build-check.sh`.

## Границы

Резерв не заменяет реальную отправку Web3Forms и не подтверждает получение заявки. Он нужен только для понятного и безопасного поведения страницы при проблемах JavaScript.
