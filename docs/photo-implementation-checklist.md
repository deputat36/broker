# Внедрение фотографии Татьяны

## Результат

Для сайта подготовлены облегчённые изображения из уже утверждённого портрета, который ранее находился внутри `assets/img/tatyana-hero.svg`.

Внешность человека, поза и содержание фотографии не генерировались и не изменялись. Выполнены только извлечение исходного изображения, масштабирование, кадрирование для аватара, сжатие и оформление Open Graph-превью.

## Файлы

В `assets/img/` находятся:

- `tatyana-hero.webp` — основной портрет 360×450, около 7 КБ;
- `tatyana-hero-mobile.webp` — мобильная версия 288×360, около 10 КБ;
- `tatyana-avatar.webp` — квадратный портрет 320×320, около 12 КБ;
- `tatyana-social.jpg` — Open Graph-превью 1200×630, около 11 КБ;
- `tatyana-hero.svg` — прежний исходный контейнер с фотографией, сохранённый для происхождения материала и аварийного восстановления.

## Подключение

Перед Jekyll-сборкой `scripts/prepare-jekyll-source.py` детерминированно:

1. заменяет hero-файл главной страницы на `tatyana-hero.webp`;
2. добавляет мобильный `<source>` с `tatyana-hero-mobile.webp`;
3. переводит страницу `/o-brokere/` на адаптивный `<picture>`;
4. добавляет responsive preload для фото первого экрана.

Скрипт работает fail-closed: если ожидаемая исходная разметка изменилась и безопасная замена больше не может быть выполнена, подготовка завершится ошибкой вместо молчаливой публикации старого варианта.

## Open Graph и Schema.org

`_layouts/default.html` использует `assets/img/tatyana-social.jpg` для:

- `og:image`;
- `og:image:secure_url`;
- `twitter:image`;
- изображения Татьяны в общем JSON-LD.

Страница `/o-brokere/` также указывает этот файл в разметке `ProfilePage`.

## Производительность и доступность

Обязательные условия:

- портрет первого экрана имеет фиксированные `width="360"` и `height="450"`;
- используется `decoding="async"`;
- первый hero-портрет получает `loading="eager"` и `fetchpriority="high"`;
- alt-текст: `Татьяна Стерликова, ипотечный брокер`;
- большой base64-JPEG больше не является основным изображением клиентских страниц;
- мобильный браузер получает отдельную уменьшенную версию.

## Автоматическая проверка

Pull request должен пройти:

- синтаксическую проверку Python;
- подготовку Jekyll source;
- Jekyll build;
- проверку существования изображений и alt-текстов;
- полный post-build QA;
- загрузку Pages-артефакта.

Дополнительно в собранном артефакте нужно проверить сигнатуры файлов:

- WebP начинается с `RIFF` и содержит `WEBP`;
- JPEG начинается с `FF D8` и завершается `FF D9`;
- HTML главной и `/o-brokere/` содержит `<picture>` и обе WebP-ссылки;
- `tatyana-social.jpg` имеет размер 1200×630.

## Ручная приёмка

После публикации остаётся проверить:

- кадрирование лица и плеч на десктопе;
- отображение портрета на реальном мобильном устройстве;
- отсутствие скачка hero-блока при загрузке;
- превью ссылки во ВКонтакте, Telegram или MAX;
- отсутствие перекрытия фотографии нижней sticky-панелью.

Эти пункты относятся к ручной приёмке опубликованного сайта и не считаются выполненными только по успешной CI-сборке.
