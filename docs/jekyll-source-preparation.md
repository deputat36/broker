# Проверка канонических Jekyll-исходников

## Назначение

Перед source-аудитами и Jekyll build workflow запускает:

```bash
bash scripts/prepare-jekyll-source.sh
```

Несмотря на историческое имя, wrapper больше не подготавливает и не переписывает файлы. Он только проверяет, что репозиторий уже содержит точную версию, которую должен собрать Jekyll.

## Что проверяется

Последовательно выполняются:

1. `audit-canonical-layout.py` — общий layout, условные ресурсы анкеты, WebP preload и безопасный thank-you parser;
2. `audit-canonical-prepared-sources.py` — privacy-safe JavaScript, WebP-фото и связанные audit-маркеры;
3. `prepare-jekyll-source.py` — канонический формат текстовых полей YAML front matter.

Любое несовпадение останавливает сборку до Jekyll.

## Front matter

Поля `layout`, `title`, `description`, `permalink`, `breadcrumb`, `h1`, Open Graph-настройки, `robots` и `hero_photo` должны храниться в кавычках либо в явном структурном YAML-формате.

Это исключает неоднозначный разбор двоеточий с пробелом в Psych. Скрипт больше не исправляет значение автоматически: он показывает файл, строку и поле и завершает CI ошибкой.

## Неизменность рабочего дерева

`scripts/check-jekyll-preparation.sh`:

- сравнивает SHA-256 Git diff до и после повторной проверки;
- выполняет `git diff --check`;
- требует подтверждения layout, privacy/photo-исходников и front matter;
- завершает CI ошибкой, если проверка изменила хотя бы один отслеживаемый файл.

Source-audit и production build используют один wrapper, поэтому между двумя путями сборки нет скрытых преобразований.

## Что удалено

Prebuild больше не выполняет:

- замену SVG-портрета на WebP;
- минимизацию localStorage;
- изменение query redirect на fragment redirect;
- условное оборачивание ресурсов анкеты;
- замену маркеров в аудитах доставки;
- автоматическое добавление кавычек во front matter.

Все эти результаты хранятся непосредственно в исходниках и защищены fail-closed аудитами.

## Правило дальнейших изменений

Новые страницы должны сразу добавляться в каноническом формате. Runtime- или prebuild-подмена допустима только как временная миграция в отдельной ветке и должна быть удалена до merge.
