# Privacy рекламной атрибуции

## Проблема

Рекламная атрибуция хранится в браузере до 90 дней с последнего посещения. Поэтому в localStorage нельзя записывать полный URL с произвольными query-параметрами или fragment: в ссылке случайно могут оказаться телефон, имя, номер обращения или другие данные, не относящиеся к рекламе.

Legacy-запись без корректного срока также не должна наследоваться бессрочно при каждом новом визите.

## Безопасный формат

Ключ localStorage: `sterlikovaMortgageTracking`.

Текущий формат имеет `storage_version: 2` и содержит:

- `first_touch` и `last_touch`;
- URL страницы только как `origin + path`;
- внутренний referrer только как `origin + path`;
- внешний referrer только как `origin`;
- заголовок страницы с ограничением длины;
- рекламные значения только из фиксированного `TRACKING_KEYS`;
- `stored_at` и `expires_at`.

Query-параметры, fragment, credentials URL и неизвестные рекламные ключи в локальную запись не попадают.

## Fail-closed чтение

Существующая запись удаляется и не наследуется, если:

- это не обычный JSON-объект;
- отсутствует `storage_version: 2`;
- `stored_at` или `expires_at` не являются корректными датами;
- срок уже истёк;
- срок установлен дальше разрешённых 90 дней;
- время записи находится в будущем за пределами небольшого допуска часов браузера;
- отсутствуют корректные `first_touch` или `last_touch`.

После очистки создаётся новая запись только из текущего безопасного контекста.

## Граница изменения

Это правило относится к 90-дневной локальной рекламной атрибуции. Поля формы, которые пользователь явно проверяет и отправляет после согласия, регулируются отдельным контрактом онлайн-заявки и политикой обработки данных.

## Автоматическая проверка

```bash
node scripts/test-tracking-privacy.js assets/js/main.js
```

Тест запускает `main.js` в изолированном mock-браузере и проверяет:

- URL с `phone`, неизвестным параметром и fragment;
- внешний referrer с query и приватным path;
- legacy-запись без версии;
- корректную запись версии 2;
- объединение только allowlist UTM;
- удаление записи с искусственно завышенным сроком;
- отсутствие контрольных секретных значений в сериализованном localStorage.

Тот же тест запускается в обязательном post-build на JavaScript из готового Pages-артефакта.
