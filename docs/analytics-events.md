# Карта целей аналитики

Источники событий: `assets/js/main.js` и `assets/js/online-application.js`.

## События для Яндекс Метрики

| Цель | Что означает | Где срабатывает |
| --- | --- | --- |
| `phone_click` | Клик по телефону | Любая ссылка `tel:` |
| `vk_click` | Переход во ВКонтакте | Ссылки на `vk.com/tatyanasterlikova` |
| `max_copy` | Нажатие на копирование номера для MAX | Кнопки с `data-copy-phone` |
| `consultation_click` | Переход на консультацию | Ссылки на `/konsultaciya/` |
| `contacts_click` | Переход на контакты | Ссылки на `/kontakty/` |
| `calculator_input` | Первое взаимодействие с калькулятором | Первое изменение поля в форме `data-mortgage-calc` |
| `online_application_click` | Переход к онлайн-заявке | Любая ссылка на `/online-zayavka/` |
| `online_application_prefill` | Форма открыта с определённым источником | Параметр `source` или внутренний referrer позволил подставить источник |
| `online_application_start` | Пользователь начал заполнять онлайн-заявку | Первое изменение поля формы `data-online-application` |
| `online_application_validation_error` | Попытка подготовить неполную заявку | Отправка формы без обязательных полей или согласия |
| `online_application_prepare` | Структурированный текст заявки успешно сформирован | Успешная отправка клиентской формы |
| `online_application_endpoint_ready` | На странице доступен включённый HTTPS endpoint | Инициализация формы после будущей активации `mode: direct` |
| `online_application_direct_attempt` | Началась прямая передача JSON | Кнопка «Отправить заявку онлайн» |
| `online_application_direct_success` | Endpoint вернул успешный HTTP-ответ | Заявка принята подключённым обработчиком |
| `online_application_direct_error` | Endpoint вернул ошибку или запрос завершился сбоем | Ошибка прямой отправки без таймаута |
| `online_application_direct_timeout` | Endpoint не ответил за установленное время | Срабатывание AbortController по таймауту |
| `online_application_spam_block` | Клиентская антиспам-проверка заблокировала прямую отправку | Заполненный honeypot, отсутствующее или слишком короткое время заполнения |
| `online_application_copy` | Пользователь скопировал готовую заявку | Кнопка «Скопировать текст» |
| `online_application_share` | Заявка передана в системное меню «Поделиться» | Успешный вызов Web Share API |
| `online_application_sms` | Пользователь перешёл к отправке заявки по SMS | Кнопка «Отправить SMS» |
| `online_application_vk` | Пользователь открыл ВКонтакте после подготовки заявки | Кнопка «Открыть ВКонтакте» |

## Атрибуция источника

Ссылки с услуг и региональных страниц передают параметр `source`. Форма записывает путь исходной страницы в скрытое поле `source_page`, использует его для предзаполнения сценария, объекта и региона, а затем добавляет строку «Источник обращения» в готовый текст и JSON заявки.

Если параметра `source` нет, скрипт пытается определить внутреннюю страницу перехода через `document.referrer`. Внешние домены не принимаются как источник формы.

Это позволяет различать обращения со страниц подбора ипотеки, отказов, семейной ипотеки, строительства дома и региональных посадочных страниц до и после подключения CRM.

## Текущий режим формы

Прямая серверная доставка отключена в `_config.yml`:

```yaml
lead_capture:
  mode: "disabled"
  endpoint: ""
```

В текущем режиме браузер только формирует текст обращения. Пользователь самостоятельно выбирает способ передачи: системное меню «Поделиться», SMS, копирование текста, ВКонтакте или MAX.

Код прямой доставки подготовлен, но кнопка остаётся скрытой, пока одновременно не выполнены два условия:

- `mode` установлен в `direct`;
- указан корректный HTTPS endpoint.

Порядок безопасного подключения описан в `docs/lead-endpoint-contract.md`.

## Как интерпретировать воронку

`online_application_prepare` означает готовую к отправке заявку, но не подтверждает доставку.

После будущей активации endpoint только `online_application_direct_success` будет означать, что обработчик вернул успешный ответ. Даже это событие должно сверяться с фактической записью в CRM, Telegram, базе или почте.

Для текущей оценки завершённых обращений учитывайте:

- `online_application_share`;
- `online_application_sms`;
- `online_application_vk`;
- `online_application_copy`;
- `phone_click`;
- `max_copy`.

Ошибки `direct_error` и `direct_timeout` не означают потерю заявки: резервные кнопки и готовый текст остаются доступны.

## Как работает кнопка MAX

Событие `max_copy` отправляется при нажатии на кнопку с `data-copy-phone`. Если браузер не поддерживает Clipboard API, пользователю показывается номер телефона, а событие всё равно фиксирует намерение связаться через MAX.

## Где задаётся ID счётчика

Файл: `_config.yml`

```yaml
yandex_metrika_id: ""
```

Когда ID пустой, код Метрики не выводится и цели не отправляются.

## Что проверить после подключения Метрики

1. ID счётчика вставлен в `_config.yml`.
2. Сайт успешно задеплоился.
3. Код Метрики появился в HTML.
4. Все цели созданы в интерфейсе Метрики.
5. Клики, ввод в калькулятор и этапы онлайн-заявки видны в отчётах.
6. Параметр `source` сохраняется в подготовленном тексте заявки.
7. `online_application_prepare` не интерпретируется как гарантированно доставленная заявка.
8. После включения endpoint отдельно проверяются success, error, timeout и fallback-сценарии.
