# Карта целей аналитики

Источник событий: `assets/js/main.js`.

## События для Яндекс Метрики

| Цель | Что означает | Где срабатывает |
| --- | --- | --- |
| `phone_click` | Клик по телефону | Любая ссылка `tel:` |
| `vk_click` | Переход во ВКонтакте | Ссылки на `vk.com/tatyanasterlikova` |
| `max_copy` | Нажатие на копирование номера для MAX | Кнопки с `data-copy-phone` |
| `consultation_click` | Переход на консультацию | Ссылки на `/konsultaciya/` |
| `contacts_click` | Переход на контакты | Ссылки на `/kontakty/` |
| `calculator_input` | Первое взаимодействие с калькулятором | Первое изменение поля в форме `data-mortgage-calc` |

## Как работает кнопка MAX

Событие `max_copy` отправляется при нажатии на кнопку с `data-copy-phone`. Если браузер не поддерживает Clipboard API, пользователю показывается номер телефона, а событие все равно фиксирует намерение связаться через MAX.

## Где задается ID счетчика

Файл: `_config.yml`

```yaml
yandex_metrika_id: ""
```

Когда ID пустой, код Метрики не выводится и цели не отправляются.

## Что проверить после подключения

1. ID счетчика вставлен в `_config.yml`.
2. Сайт успешно задеплоился.
3. Код Метрики появился в HTML.
4. Все цели созданы в интерфейсе Метрики.
5. Клики и ввод в калькулятор видны в отчетах.
