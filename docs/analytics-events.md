# Карта целей аналитики

Источники событий: `assets/js/main.js` и `assets/js/online-application.js`.

## События для Яндекс Метрики

| Цель | Что означает | Где срабатывает |
| --- | --- | --- |
| `phone_click` | Клик по телефону | Любая ссылка `tel:` |
| `vk_click` | Переход во ВКонтакте | Ссылки на `vk.com/tatyanasterlikova` |
| `max_copy` | Нажатие на копирование номера для MAX | Кнопки с `data-copy-phone` |
| `consultation_click` | Переход на консультацию | Ссылки на `/konsultaciya/` |
| `contacts_click` | Переход на контакты | Ссылки на `/kontakty/` |
| `calculator_input` | Первое взаимодействие с калькулятором | Первое изменение поля в форме `data-mortgage-calc` |
| `online_application_start` | Пользователь начал заполнять онлайн-заявку | Первое изменение поля формы `data-online-application` |
| `online_application_validation_error` | Попытка подготовить неполную заявку | Отправка формы без обязательных полей или согласия |
| `online_application_prepare` | Структурированный текст заявки успешно сформирован | Успешная отправка клиентской формы |
| `online_application_copy` | Пользователь скопировал готовую заявку | Кнопка «Скопировать текст» |
| `online_application_share` | Заявка передана в системное меню «Поделиться» | Успешный вызов Web Share API |
| `online_application_sms` | Пользователь перешёл к отправке заявки по SMS | Кнопка «Отправить SMS» |
| `online_application_vk` | Пользователь открыл ВКонтакте после подготовки заявки | Кнопка «Открыть ВКонтакте» |

## Как работает онлайн-заявка

Статическая форма не отправляет сведения в фоновом режиме и не использует неподтвержденный серверный обработчик. После валидации браузер формирует текст обращения. Пользователь самостоятельно выбирает способ передачи: системное меню «Поделиться», SMS, копирование текста, ВКонтакте или MAX.

Цель `online_application_prepare` означает готовую к отправке заявку, но не подтверждает, что сообщение фактически доставлено Татьяне. Для оценки завершенных обращений нужно дополнительно учитывать `online_application_share`, `online_application_sms`, `online_application_vk`, звонки и копирование номера MAX.

## Как работает кнопка MAX

Событие `max_copy` отправляется при нажатии на кнопку с `data-copy-phone`. Если браузер не поддерживает Clipboard API, пользователю показывается номер телефона, а событие все равно фиксирует намерение связаться через MAX.

## Где задается ID счетчика

Файл: `_config.yml`

```yaml
yandex_metrika_id: ""
```

Когда ID пустой, код Метрики не выводится и цели не отправляются.

## Что проверить после подключения

1. ID счетчика вставлен в `_config.yml`.
2. Сайт успешно задеплоился.
3. Код Метрики появился в HTML.
4. Все цели созданы в интерфейсе Метрики.
5. Клики, ввод в калькулятор и этапы онлайн-заявки видны в отчетах.
6. Цель `online_application_prepare` не интерпретируется как гарантированно доставленная заявка.
