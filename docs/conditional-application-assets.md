# Условная загрузка ресурсов онлайн-заявки

## Назначение

Основной контроллер и стили анкеты нужны только на странице `/online-zayavka/`. Их глобальное подключение заставляло остальные страницы загружать код формы, хотя на них отсутствовал элемент `data-online-application`.

## Какие ресурсы ограничены

- `/assets/css/online-application.css`;
- `/assets/js/online-application.js`.

Вспомогательные модули `application-inputs.js`, `application-preparation.js`, `application-delivery-keepalive.js` и `thankyou-storage-privacy.js` уже подключаются непосредственно в `online-zayavka.md` и не являются глобальными.

## Как работает подготовка

`scripts/prepare-conditional-assets.py` выполняет точную fail-closed замену двух глобальных тегов в `_layouts/default.html` на Liquid-условия для `page.url == '/online-zayavka/'`.

Скрипт:

- не меняет клиентскую логику анкеты;
- не переименовывает asset-файлы;
- допускает повторный запуск без изменений;
- останавливает сборку, если ожидаемая структура layout изменилась.

Общий wrapper `scripts/prepare-jekyll-source.sh` запускает основную подготовку Jekyll и условную загрузку ресурсов в одинаковом порядке для source-аудита и production build.

## Автоматическая проверка

`scripts/audit-conditional-application-assets.py` проверяет готовый `_site`:

- CSS и JavaScript анкеты существуют и не пусты;
- на `/online-zayavka/` каждый ресурс подключён ровно один раз;
- на остальных HTML-страницах эти ресурсы отсутствуют;
- форма `data-online-application` находится только на странице анкеты.

Проверка подключена к обязательному `post-build-check.sh`.

## Ручная приёмка

После публикации необходимо отдельно проверить:

1. открытие `/online-zayavka/` без ошибок в консоли;
2. отображение всех шагов и дополнительных полей;
3. подготовку, копирование и отправку обращения;
4. работу fallback-каналов;
5. отсутствие запросов к `online-application.css` и `online-application.js` на главной и обычных информационных страницах.

## Ограничения

Условная загрузка сокращает вес обычных страниц, но не уменьшает общий размер Pages-артефакта: файлы анкеты продолжают публиковаться и используются на одном маршруте. Реальные сетевые показатели подтверждаются браузерной проверкой и Core Web Vitals.
