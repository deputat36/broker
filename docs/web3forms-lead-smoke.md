# Smoke-тест приёма заявок через Web3Forms

## Назначение

Этот чек-лист подтверждает не только работу JavaScript, но и фактическую доставку заявки на правильный email.

Схема аналогична проекту `deputat36/bm`:

1. пользователь заполняет форму;
2. Web3Forms отправляет email-копию;
3. сайт показывает `/spasibo/`;
4. при ошибке остаются SMS, MAX, ВКонтакте, Web Share и копирование текста;
5. будущий Supabase endpoint может работать параллельно в режиме `hybrid`.

## Тестовая ссылка

Открыть форму с тестовой атрибуцией:

```text
https://sterlikova-ipoteka.ru/online-zayavka/?source=%2Fuslugi%2Fpodbor-ipoteki%2F&utm_source=smoke&utm_medium=test&utm_campaign=web3forms_launch&utm_content=desktop
```

Для мобильной проверки заменить `utm_content=desktop` на `utm_content=mobile`.

## Тестовые данные

Использовать явно тестовые значения:

- имя: `Тест Web3Forms`;
- телефон: рабочий тестовый номер владельца проекта;
- город: `Борисоглебск`;
- сценарий: `Первичная консультация и подбор ипотеки`;
- объект: `Пока не выбрано`;
- стоимость: `Тест`;
- первоначальный взнос: `Тест`;
- история банков: `Тестовая заявка, не обрабатывать как клиента`;
- комментарий: `SMOKE TEST. После проверки письмо удалить.`

Не использовать чужой номер, паспортные данные, СНИЛС, банковские реквизиты или реальные кредитные сведения.

## Успешный сценарий

1. Заполнить обязательные поля.
2. Установить чекбокс согласия.
3. Нажать «Проверить и подготовить заявку».
4. Проверить сформированный текст:
   - присутствует request ID;
   - источник `/uslugi/podbor-ipoteki/`;
   - указан город;
   - указан сценарий;
   - нет паспортных или банковских документов.
5. Нажать «Отправить заявку онлайн».
6. Убедиться, что кнопка переходит в состояние отправки и защищена от повторного клика.
7. Дождаться перехода на `/spasibo/`.
8. Проверить на странице благодарности:
   - номер обращения совпадает;
   - сценарий совпадает;
   - город отображается;
   - статус квалификации отображается;
   - номер телефона не выводится.

## Проверка email

В рабочем почтовом ящике проверить:

- [ ] письмо поступило;
- [ ] определён фактический адрес получателя Web3Forms;
- [ ] письмо не попало в спам;
- [ ] тема начинается с `Новая заявка ипотечному брокеру`;
- [ ] присутствуют имя, телефон и город;
- [ ] присутствуют сценарий, объект, стоимость и взнос;
- [ ] присутствуют request ID и страница-источник;
- [ ] присутствуют `utm_source=smoke`, `utm_medium=test`, `utm_campaign=web3forms_launch`;
- [ ] присутствуют квалификация и приоритет;
- [ ] присутствует дата отправки;
- [ ] письмо читается на телефоне и компьютере.

Если письмо пришло не тому получателю, приём заявок нельзя считать введённым в эксплуатацию. Нужно создать отдельный Web3Forms access key для подтверждённого рабочего email и заменить ключ в `assets/js/online-application.js`.

## Проверка повторной отправки

- [ ] Во время одного запроса повторный клик заблокирован.
- [ ] После успешной отправки повторный клик невозможен из-за перехода на `/spasibo/`.
- [ ] Новая заявка получает другой request ID.
- [ ] На странице благодарности не сохраняется телефон.

Web3Forms сам по себе не обеспечивает бизнес-дедупликацию. Постоянная идемпотентность по request ID будет реализована в Supabase backend v2.

## Проверка ошибки

Ошибку проверять только в тестовой ветке или локально, временно подменив URL Web3Forms на недоступный адрес.

Ожидаемое поведение:

- [ ] появляется понятное сообщение об ошибке;
- [ ] страница не переходит на `/spasibo/`;
- [ ] готовый текст остаётся в поле;
- [ ] работает копирование;
- [ ] доступно SMS;
- [ ] открывается ВКонтакте;
- [ ] можно скопировать номер для MAX;
- [ ] работает системное меню «Поделиться», если поддерживается браузером.

После проверки тестовую подмену обязательно удалить.

## Проверка таймаута

При искусственно зависшем запросе через время из `lead_capture.timeout_ms` должны выполниться условия:

- [ ] запрос отменён через `AbortController`;
- [ ] показано сообщение о таймауте;
- [ ] кнопка позволяет повторить отправку;
- [ ] резервные способы остаются доступными.

## Проверка антиспама

- [ ] Honeypot скрыт от обычного пользователя.
- [ ] Заполненный honeypot блокирует онлайн-отправку.
- [ ] Слишком быстрая отправка блокируется клиентской проверкой.
- [ ] Блокировка не показывает злоумышленнику внутренние детали.

Клиентский антиспам не заменяет server-side rate limit. Он будет добавлен при подключении Supabase в режиме `hybrid`.

## Проверка аналитики

После подключения Яндекс Метрики проверить цели:

- `online_application_click`;
- `online_application_prefill`;
- `online_application_start`;
- `online_application_prepare`;
- `online_application_direct_attempt`;
- `online_application_direct_success`;
- `lead_submit`;
- `lead_thankyou_view`.

`lead_submit` означает успешный технический ответ Web3Forms. Фактическую доставку подтверждает наличие письма в рабочем ящике.

## Результат запуска

Зафиксировать:

- дата и время теста;
- устройство и браузер;
- request ID;
- адрес получателя;
- время доставки письма;
- папка, куда поступило письмо;
- результат перехода на `/spasibo/`;
- результат fallback-теста;
- найденные ошибки;
- ответственный за обработку реальных заявок.

После успешного теста удалить тестовое письмо и не использовать тестовую запись как реального клиента.
