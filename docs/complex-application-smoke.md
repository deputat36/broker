# Smoke-тест сложного маршрута онлайн-заявки

## Цель

Проверить цепочку:

`региональная страница → маршрут подготовки → предзаполненная форма → отметки клиента → готовый текст → Web3Forms`.

Тест дополняет `docs/web3forms-lead-smoke.md` и не заменяет подтверждение фактического email-получателя.

## Тестовые страницы

Проверить минимум по одной странице каждого сценария:

- `/geo/borisoglebsk/otkazali-v-ipoteke/`;
- `/geo/gribanovskiy/ipoteka-s-plohoy-kreditnoy-istoriey/`;
- `/geo/povorino/ipoteka-bez-oficialnogo-dohoda/`;
- `/geo/gribanovskiy/ipoteka-bez-pervonachalnogo-vznosa/`.

## Переход к форме

На каждой странице:

1. Найти блок «От диагностики к заявке».
2. Проверить наличие четырёх шагов.
3. Перейти по четвёртому шагу.
4. Убедиться, что URL формы содержит:
   - `source` с точным адресом исходной страницы;
   - `journey=complex`;
   - `stage=route`.

## Предзаполнение

После открытия `/online-zayavka/` проверить:

- [ ] город или район подставлен из страницы-источника;
- [ ] сценарий соответствует исходной странице;
- [ ] появился блок «Что уже успели проверить»;
- [ ] блок «Добавить подробности» получил номер шага 3;
- [ ] при обычном открытии `/online-zayavka/` блок подготовки скрыт;
- [ ] четыре подписи меняются в зависимости от сценария.

## Сценарные подписи

### После отказа банка

Должны упоминаться:

- банк, дата и этап отказа;
- кредиты, карты и платежи;
- объект и запросы банка;
- отсутствие новых хаотичных заявок.

### Плохая кредитная история

Должны упоминаться:

- получение или проверка кредитной истории;
- действующие обязательства и лимиты;
- просрочки, закрытые долги или ошибки;
- отсутствие массовой подачи в банки.

### Нестандартный доход

Должны упоминаться:

- реальные источники дохода;
- доход, кредиты и комфортный платеж;
- подтверждения регулярных поступлений;
- отказ от недостоверных сведений и документов.

### Нет первоначального взноса

Должны упоминаться:

- недостающая сумма;
- маткапитал, накопления или продажа имущества;
- расходы после покупки и резерв;
- отказ от фиктивного завышения стоимости.

## Подготовка тестовой заявки

Использовать явно тестовые значения:

- имя: `Тест сложного маршрута`;
- телефон: подтверждённый тестовый номер владельца;
- город: оставить предзаполненное значение;
- отметить две выполненные проверки;
- в поле оставшихся вопросов написать: `SMOKE TEST — нужно уточнить следующий шаг`;
- в обычном комментарии написать: `Тест контекста подготовки. Не обрабатывать как реального клиента.`

После нажатия «Проверить и подготовить заявку» в тексте должны присутствовать:

- [ ] точная страница-источник;
- [ ] правильный ипотечный сценарий;
- [ ] `Этап подготовки: После изучения маршрута подготовки`;
- [ ] строка `Что уже проверено` с двумя выбранными пунктами;
- [ ] строка `Что осталось уточнить`;
- [ ] отдельный заголовок `Комментарий клиента`;
- [ ] предупреждение, что решение принимает банк.

В видимом поле обычного комментария после подготовки не должен оставаться автоматически добавленный служебный блок.

## Проверка доставки

После отправки через Web3Forms проверить письмо:

- [ ] письмо дошло подтверждённому получателю;
- [ ] в поле комментария присутствует контекст подготовки;
- [ ] исходный комментарий клиента не потерян;
- [ ] источник и request ID присутствуют;
- [ ] текст не обрезан посередине ключевой строки;
- [ ] нет паспортных данных, документов и банковских реквизитов;
- [ ] произошёл переход на `/spasibo/`.

## Аналитика

После подключения Яндекс Метрики проверить:

- `online_application_click`;
- `online_application_prefill`;
- `online_application_complex_prefill`;
- `online_application_preparation_check`;
- `online_application_prepare`;
- `online_application_direct_success`;
- `lead_submit`;
- `lead_thankyou_view`.

## Мобильная проверка

На экране 360–430 px убедиться:

- [ ] четыре отметки отображаются в одну колонку;
- [ ] подписи не обрезаются;
- [ ] textarea оставшихся вопросов доступна без горизонтальной прокрутки;
- [ ] согласие и кнопка отправки не перекрываются липкой панелью;
- [ ] после подготовки доступны основной и резервные каналы.

## Результат

Зафиксировать:

- дату и устройство;
- исходную региональную страницу;
- сценарий;
- выбранные отметки;
- request ID;
- адрес получателя;
- результат доставки;
- результат `/spasibo/`;
- найденные расхождения.

После теста удалить письмо и не учитывать заявку как реальное обращение.
