# Дизайн-система сайта ипотечного брокера — 2026

Figma-файл: https://www.figma.com/design/Sz62KR5VogV6KAGJfeHCZi

Статус Figma: отдельный файл создан. Дальнейшее редактирование canvas временно ограничено лимитом Figma MCP Starter. Источником истины для внедрённой версии до восстановления доступа является этот документ и дизайн-слой в `assets/css/nav-state.css`.

## 1. Концепция

Сайт должен восприниматься не как типовой банковский лендинг, а как цифровой кабинет личного ипотечного специалиста.

Основные ощущения:

- спокойствие вместо давления;
- профессиональная уверенность без обещаний одобрения;
- персональное сопровождение вместо обезличенного сервиса;
- премиальность без демонстративной роскоши;
- удобство для клиента из любого города;
- одинаковое доверие на desktop и mobile.

## 2. Визуальное направление

### Палитра

| Роль | Значение | CSS |
|---|---|---|
| Основной тёмный | глубокий синий | `#17283D` |
| Вторичный | спокойный зелёный | `#2F6B62` |
| Акцент | тёплое золото | `#C89445` |
| Тёплый фон | слоновая кость | `#FBF7EF` |
| Холодный фон | светлый серо-зелёный | `#F3F6F6` |
| Основной текст | сине-графитовый | `#29394D` |
| Вторичный текст | нейтральный серый | `#687588` |
| Границы | холодный светло-серый | `#DCE3E7` |

Красный цвет исключается из новой идентичности. Старые CSS-переменные `--red`, `--red-dark` и `--red-soft` временно используются как совместимые aliases для зелёной семантики, чтобы не переписывать 123 страницы одномоментно.

### Типографика

Используется системный стек без внешней загрузки:

```css
Inter, "Segoe UI", Arial, sans-serif
```

Правила:

- заголовки — плотные, с отрицательным tracking;
- основной вес заголовков 850–900;
- основной текст не мельче 16 px;
- лид-абзацы 19–24 px;
- eyebrow — uppercase, увеличенный tracking, золотой акцент;
- длина строки основного контента ограничивается существующей сеткой.

### Геометрия

- базовый радиус: 18 px;
- малый радиус: 12 px;
- hero-card: 26 px desktop / 20 px mobile;
- кнопки: 12 px;
- карточки не вращаются;
- тени мягкие, с холодным синим оттенком;
- декоративные линии используют градиент золото → зелёный.

## 3. Компоненты

### Header

- кремовый полупрозрачный фон;
- фотография Татьяны в золотом кольце;
- активный раздел выделяется мягким зелёным фоном;
- онлайн-заявка получает золотой фон;
- телефон остаётся тёмной контрастной кнопкой.

### Hero

- глубокий синий фон с зелёным переходом;
- мягкое золотое свечение;
- фотография Татьяны — главный доверительный объект;
- никаких наклонённых карточек;
- первичный CTA — золотой;
- альтернативные действия — стеклянные светлые кнопки;
- статистика и факты оформляются полупрозрачными блоками.

### Карточки

- белая поверхность;
- тонкая холодная граница;
- верхняя линия золото → зелёный;
- радиус 18 px;
- мягкое поднятие при hover;
- без тяжёлых цветных фонов, кроме featured-сценариев.

### Формы

- белые поля;
- радиус 12 px;
- focus-ring спокойного зелёного цвета;
- результаты калькулятора — тёмно-синие с зелёным переходом;
- логика, validation и payload не меняются дизайном.

### CTA и notice

- тёмно-синий фон с зелёным переходом;
- золотое декоративное свечение;
- основной CTA — золотой;
- вторичные CTA — прозрачные или зелёные.

### Footer

- глубокий синий → тёмно-зелёный;
- фотография Татьяны с золотой границей;
- активные ссылки и ссылка «О подходе» — тёплое золото;
- дисклеймер отделён тонкой линией.

## 4. Адаптивность

### Desktop

- максимальная рабочая ширина: 1200 px;
- header: до 1320 px;
- hero — две колонки;
- ключевые сетки: 3–4 колонки по существующим классам.

### Tablet

- меню переходит в раскрывающуюся панель;
- hero и split-блоки переходят в одну колонку;
- карточки — две колонки.

### Mobile

- горизонтальные поля: 18 px;
- hero начинается сразу после компактного header;
- кнопки сохраняют текущую полноширинную механику;
- нижняя панель содержит четыре действия в одну строку;
- минимальная высота интерактивных элементов: 44 px.

## 5. Карта Figma

План страниц Figma-файла:

1. `00 Cover` — позиционирование и принципы;
2. `01 Foundations` — цвета, типографика, радиусы, тени, spacing;
3. `02 Components` — кнопки, карточки, формы, header, footer;
4. `03 Homepage Desktop` — главная 1440 px;
5. `04 Homepage Mobile` — главная 390 px;
6. `05 Inner Pages` — услуга, статья, FAQ, контакты, заявка;
7. `06 States` — hover, focus, error, success, mobile menu.

## 7. Ограничения

Дизайн не должен:

- менять тарифы и условия компании «ЭТАЖИ»;
- добавлять отзывы, достижения или цифры без подтверждения;
- обещать одобрение ипотеки;
- изменять Web3Forms/Supabase payload;
- увеличивать JavaScript;
- скрывать политику обработки данных;
- ухудшать keyboard navigation и focus states;
- нарушать существующие contextual CTA.

## 8. Следующие этапы

После восстановления лимита Figma MCP:

1. перенести foundations в variables и styles;
2. создать компоненты Button, Card, Input, Header, CTA;
3. собрать desktop/mobile главную;
4. добавить макеты внутренних страниц;
5. проверить соответствие уже внедрённому CSS;
6. при необходимости синхронизировать уточнения обратно в GitHub отдельным PR.
