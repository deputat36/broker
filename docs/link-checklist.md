# Чек-лист проверки ссылок

Документ нужен после добавления новых страниц, статей и SEO-разделов.

## Что проверять в первую очередь

- Главное меню.
- Подвал сайта.
- Sticky-панель связи.
- Страницу `/uslugi/`.
- Страницу `/geo/`.
- Страницу `/polezno/`.
- Страницу `/konsultaciya/`.
- Страницу `/kontakty/`.
- `sitemap.xml`.

## Проверка новой страницы

Для каждой новой страницы проверить:

- страница открывается по permalink;
- есть входящая ссылка с хаба или близкой страницы;
- есть ссылка на консультацию или контакт;
- есть 2–3 внутренние ссылки на близкие материалы;
- страница добавлена в `sitemap.xml`, если должна индексироваться;
- нет ссылок на несуществующие URL;
- внешние ссылки ведут только на проверенные страницы.

## Частые места ошибок

- Новый материал создан, но не добавлен в хаб.
- URL добавлен в sitemap, но на него нет входящей ссылки.
- Ссылка написана без завершающего `/`.
- Внутренняя ссылка ведет на старый slug.
- В карточке есть ссылка, но заголовок визуально выглядит как обычный текст.

## Быстрая ручная приемка

После деплоя открыть сайт и пройти путь пользователя:

1. Главная.
2. Услуги.
3. География.
4. Полезная статья.
5. Калькулятор.
6. Консультация.
7. Контакты.
8. Телефон, MAX, ВК.

Если весь путь проходит без тупиков, базовая перелинковка работает.
