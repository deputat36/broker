# Подключение домена через Beget

Инструкция для запуска сайта GitHub Pages на домене `sterlikova-ipoteka.ru`.

## Что уже настроено в репозитории

В файле `_config.yml` указан основной адрес сайта:

```yml
url: "https://sterlikova-ipoteka.ru"
baseurl: ""
```

В корне репозитория есть файл `CNAME` со значением:

```text
sterlikova-ipoteka.ru
```

## DNS-записи в Beget

В панели Beget нужно открыть домен и перейти в управление DNS-зоной.

Для основного домена нужно добавить четыре A-записи:

```text
@    A    185.199.108.153
@    A    185.199.109.153
@    A    185.199.110.153
@    A    185.199.111.153
```

Для поддомена `www` нужно добавить CNAME-запись:

```text
www    CNAME    deputat36.github.io
```

## Что удалить или заменить

Если в DNS-зоне есть старые записи Beget для сайта, они могут мешать работе GitHub Pages.

Нужно убрать или заменить записи такого типа:

```text
@      A        старый IP Beget
www    A        старый IP Beget
www    CNAME    старое значение
```

MX-записи почты удалять не нужно, если почта на домене используется.

## Настройка в GitHub

В репозитории нужно открыть:

```text
Settings -> Pages -> Custom domain
```

Указать домен:

```text
sterlikova-ipoteka.ru
```

После проверки DNS включить:

```text
Enforce HTTPS
```

## Проверка после запуска

Проверить адреса:

```text
https://sterlikova-ipoteka.ru/
https://www.sterlikova-ipoteka.ru/
https://sterlikova-ipoteka.ru/sitemap.xml
https://sterlikova-ipoteka.ru/robots.txt
```

Основной домен должен открывать сайт. Версия с `www` должна открываться или перенаправляться на основной домен.

## Если сайт не открылся сразу

DNS может обновляться не мгновенно. После правок в Beget обычно нужно подождать.

Если через несколько часов сайт не открывается, проверить:

- нет ли лишних A-записей на IP Beget;
- правильно ли указан CNAME для `www`;
- добавлен ли домен в GitHub Pages;
- создан ли файл `CNAME` в корне репозитория;
- включен ли HTTPS после проверки домена.
