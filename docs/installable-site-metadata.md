# Метаданные установки сайта

## Назначение

Сайт содержит web app manifest и набор иконок для браузеров и мобильных устройств. Это позволяет корректно показывать название и фирменный знак при добавлении сайта на главный экран или установке через поддерживаемый браузер.

## Что подключено

- `site.webmanifest` с `id`, `start_url` и `scope`, равными `/`;
- режим отображения `standalone`;
- PNG 192×192 для стандартной установки;
- PNG 512×512 для экранов высокой плотности;
- отдельная maskable-иконка 512×512;
- `apple-touch-icon` 180×180;
- SVG favicon как масштабируемая иконка вкладки.

Все HTML-страницы получают manifest и Apple-метаданные через общий `_layouts/default.html`.

## Границы

Наличие manifest не означает офлайн-режим. В проекте не заявляется работа без сети и не устанавливается service worker только ради формального статуса PWA.

Фактическое предложение установки зависит от браузера, операционной системы, HTTPS, пользовательских настроек и правил конкретной платформы.

## Автоматическая проверка

```bash
python3 scripts/audit-installable-site.py ./_site
```

Аудит проверяет:

- корректный JSON manifest;
- обязательные значения `id`, `start_url`, `scope`, `display` и `lang`;
- записи PNG-иконок и их `purpose`;
- реальную PNG-сигнатуру и размеры 180, 192 и 512 пикселей;
- отдельную maskable-иконку;
- единственное подключение manifest, `apple-touch-icon` и Apple standalone-метаданных на каждой HTML-странице.

Проверка включена в обязательный post-build suite.
