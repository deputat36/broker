## 3. Основные проблемы и риски

### 3.1. Фото Татьяны

На главной странице сейчас используется временный файл:

```text
/assets/img/tatyana-hero.svg
```

Это SVG со встроенным base64-изображением. Нужно заменить на нормальные файлы:

```text
/assets/img/tatyana-hero.webp
/assets/img/tatyana-hero-mobile.webp
/assets/img/tatyana-social.jpg
/assets/img/tatyana-avatar.webp
```

Пока этого не сделать, первый экран может выглядеть хуже, фото может не отображаться стабильно, а скорость загрузки будет хуже оптимальной.

### 3.2. Метрика не подключена

В `_config.yml` сейчас:

```yml
yandex_metrika_id: ""
```

Код Метрики подготовлен, но без ID счетчика он не выводится. До запуска рекламы нужно вставить ID и проверить цели.

### 3.3. Форма заявок не включена

Supabase готов, но frontend-форма пока не работает. Это нормально для текущего безопасного состояния, но для рекламы форма может повысить конверсию.

Важно: подключать форму только после тестирования Edge Function, CORS, ошибок, согласия на обработку данных и защиты от спама.

### 3.4. Хаб полезных материалов перегружен

Страница `/polezno/` содержит много карточек подряд. Это хорошо для индексации, но может быть тяжело для пользователя. Нужно сгруппировать материалы по смысловым блокам:

- подготовка к ипотеке;
- доход и кредитная нагрузка;
- объект недвижимости;
- сложные ситуации;
- маткапитал и семейные сценарии;
- сделка и расходы.

### 3.5. README частично отстает от текущего состояния

README пишет, что формы заявок пока не используются. Это все еще верно для публичного сайта, но нужно уточнить: backend Supabase уже подготовлен, frontend не включен.

Также список полезных материалов в README неполный относительно фактического количества страниц.

### 3.6. Домен и HTTPS нужно проверить вручную

В репозитории все подготовлено, но важно проверить в GitHub Pages:

- custom domain;
- DNS check;
- Enforce HTTPS;
- открытие `robots.txt` и `sitemap.xml` на основном домене.

### 3.7. Социальное превью зависит от файла `tatyana-social.jpg`

Шаблон использует `/assets/img/tatyana-social.jpg`. Если файл отсутствует или неактуален, ссылки в мессенджерах и ВК будут выглядеть хуже.

### 3.8. Не все документы связаны в `docs/README.md`

В проекте накопилось много служебных документов. Нужно периодически актуализировать индекс документации, но делать это аккуратно, потому что большие правки иногда блокируются.

## 4. Приоритеты автономной работы

### Приоритет 1. Готовность к запуску домена и рекламы

1. Проверять и поддерживать `_config.yml`, `CNAME`, `robots.txt`, `sitemap.xml`.
2. Поддерживать инструкцию Beget/GitHub Pages.
3. После получения ID Метрики — вставить его в `_config.yml`.
4. Проверить цели в `main.js`.
5. Проверить клики по телефону, VK, MAX, калькулятору и странице консультации.
6. После загрузки фото заменить SVG на WebP.

### Приоритет 2. Фото и доверие

1. Дождаться загрузки файлов фото в `assets/img/`.
2. Переключить `index.md` с `tatyana-hero.svg` на `tatyana-hero.webp`.
3. Заменить hero-изображение на `picture` с мобильным `source`.
4. Проверить Open Graph через `tatyana-social.jpg`.
5. Добавить фото на страницу `o-brokere`, если там оно не используется полноценно.

### Приоритет 3. Конверсия

1. Усилить страницу консультации.
2. Сделать более короткий путь до звонка и MAX.
3. Добавить блоки `что подготовить перед звонком`.
4. Добавить мини-скрипты первого сообщения для MAX/VK.
5. Подготовить форму заявки только после успешного тестирования Supabase-функции.

### Приоритет 4. SEO и перелинковка

1. Проверить, что все страницы из `sitemap.xml` реально существуют.
2. Проверить, что все важные статьи присутствуют в хабе `/polezno/`.
3. Сгруппировать полезные материалы по темам.
4. Добавлять внутренние ссылки из статей на услуги, консультацию, калькулятор и смежные статьи.
5. Развивать гео-кластеры аккуратно, без дублей и переспама.

### Приоритет 5. Техническое качество

1. Уменьшать вес первого экрана.
2. Не добавлять тяжелые библиотеки.
3. Сохранять доступность: focus-visible, aria-live, понятные alt.
4. Поддерживать print.css.
5. Проверять мобильную навигацию.
6. Не ломать `main.js`, потому что он отвечает за меню, MAX, аналитику и калькулятор.
