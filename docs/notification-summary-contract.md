# Контракт серверной сводки заявки для уведомлений

## Назначение

Функция `public.broker_lead_notification_summary(uuid)` формирует ограниченный текст заявки для будущего Telegram-уведомления или внутреннего CRM-канала.

Она не отправляет сообщение самостоятельно, не хранит Telegram-токен и не выполняет HTTP-запросы. Вызов выполняется только серверным кодом после сохранения заявки.

## Миграция

Функция создаётся миграцией:

`supabase/migrations/202607130004_broker_lead_notification_summary.sql`

Она применяется после:

1. `202607070001_create_broker_leads.sql`;
2. `202607130002_broker_leads_v2.sql`;
3. `202607130003_broker_lead_preparation.sql`.

До применения всех миграций `lead_capture.endpoint` остаётся пустым, а рабочий режим сайта — `web3forms`.

## Доступ

Функция использует `security definer`, фиксированный `search_path = public` и принимает только UUID заявки.

Права:

- `public` — запрещено;
- `anon` — запрещено;
- `authenticated` — запрещено;
- `service_role` — разрешён вызов.

Функция не должна вызываться из браузера и не должна становиться публичным RPC.

## Источник данных

Сводка строится только из строки `public.broker_leads`, уже сохранённой серверным обработчиком.

Используются:

- request ID;
- имя и телефон;
- город и способ связи;
- ипотечный сценарий;
- объект, стоимость, взнос и доход;
- технический приоритет;
- страница-источник;
- очищенные поля подготовки из миграции `202607130003`.

Исходный `raw_payload` напрямую в уведомление не вставляется.

## Контекст подготовки

Раздел `ПОДГОТОВКА ДО ОБРАЩЕНИЯ` добавляется только при наличии хотя бы одного значения:

- `journey_type`;
- `journey_stage`;
- `journey_scenario_slug`;
- `preparation_completed`;
- `remaining_questions`.

Отметки берутся из очищенного JSON-массива `preparation_completed`. Используется не более четырёх значений, прошедших whitelist в trigger `sync_broker_lead_preparation`.

Пример:

```text
ПОДГОТОВКА ДО ОБРАЩЕНИЯ
Тип маршрута: Сложный региональный маршрут
Этап: После изучения маршрута подготовки
Сценарий: otkazali-v-ipoteke
Что уже проверено: Зафиксировал банк и этап отказа; Проверил кредиты и карты
Что осталось уточнить: Нужно понять, связан ли отказ с объектом
```

Эта информация не означает вероятность банковского одобрения и не заменяет проверку брокером.

## Формат результата

Базовый текст:

```text
Новая заявка ипотечному брокеру
ID: ...
Имя: ...
Телефон: ...
Город: ...
Связь: ...
Задача: ...
Объект: ...
Стоимость: ...
Взнос: ...
Доход: ...
Приоритет: ...
Источник: ...
```

Для сложной заявки ниже добавляется раздел подготовки.

## Обработка ошибок

При неизвестном UUID функция завершает вызов ошибкой `broker_lead_not_found`.

Серверный канал уведомления должен:

- не раскрывать текст ошибки клиенту сайта;
- не считать ошибку уведомления ошибкой сохранения заявки;
- перевести `notification_status` в `failed`;
- создать техническое событие без дублирования полного payload.

## Telegram

Telegram-токен и chat ID хранятся только в secrets Edge Function.

После активации Supabase обработчик должен:

1. сохранить заявку;
2. получить сводку через `broker_lead_notification_summary` от имени `service_role`;
3. отправить текст в подтверждённый Telegram-чат;
4. обновить `notification_status`;
5. записать событие `notification_sent` или `notification_failed`.

Прямое включение Telegram до проверки миграций, получателя, срока хранения и smoke-теста запрещено.
