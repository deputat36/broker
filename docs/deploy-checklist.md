# Чек-лист проверки деплоя

Документ нужен после крупных правок сайта, чтобы быстро проверить публикацию на GitHub Pages и основной домен.

## 1. Проверить workflow

Основной файл деплоя:

- `.github/workflows/pages.yml`

Что должно быть:

- запуск при push в `main`;
- возможность ручного запуска через `workflow_dispatch`;
- сборка Jekyll;
- загрузка артефакта Pages;
- деплой в GitHub Pages.

## 2. Проверить настройки Pages

В настройках репозитория:

1. Открыть `Settings`.
2. Перейти в `Pages`.
3. Проверить, что источник публикации — GitHub Actions.
4. Проверить, что custom domain указан как:

`sterlikova-ipoteka.ru`

5. Проверить, что включен `Enforce HTTPS`.
6. Проверить, что сайт открывается по основному адресу:

`https://sterlikova-ipoteka.ru/`

Запасной технический адрес GitHub Pages:

`https://deputat36.github.io/broker/`

В рекламе и внешних материалах использовать основной домен, а не технический адрес.

## 3. Проверить основные страницы после деплоя

После публикации открыть на основном домене:

- `/`;
- `/uslugi/`;
- `/konsultaciya/`;
- `/kalkulyator-ipoteki/`;
- `/stoimost/`;
- `/polezno/`;
- `/faq/`;
- `/o-brokere/`;
- `/kontakty/`;
- `/404.html`.

## 4. Проверить технические URL

Открыть напрямую:

- `/robots.txt`;
- `/sitemap.xml`;
- `/llms.txt`;
- `/humans.txt`;
- `/site.webmanifest`.

## 5. Проверить мобильную версию

На телефоне или через эмулятор проверить:

- шапку сайта;
- меню;
- первый экран;
- sticky-панель связи;
- кнопки телефона, MAX и ВК;
- ипотечный калькулятор;
- длинные SEO-страницы;
- футер.

## 6. Проверить клики и события

Проверить вручную:

- клик по телефону;
- копирование номера для MAX;
- переход во ВКонтакте;
- переход на консультацию;
- ввод данных в калькулятор.

После подключения Яндекс Метрики дополнительно проверить цели:

- `phone_click`;
- `vk_click`;
- `max_copy`;
- `consultation_click`;
- `contacts_click`;
- `calculator_input`.

## 7. Проверить SEO-основу

На нескольких страницах проверить:

- title;
- description;
- canonical;
- robots meta;
- Open Graph image;
- хлебные крошки;
- Schema.org-разметку.

## 8. Проверить печать

После добавления `assets/css/print.css` проверить печать для:

- FAQ;
- полезной статьи;
- страницы консультации;
- страницы стоимости.

В печатной версии не должны мешать:

- шапка;
- меню;
- sticky-панель;
- CTA-кнопки;
- футер.

## 9. Что делать при ошибке сборки

Если сайт не обновился:

1. Открыть вкладку `Actions`.
2. Найти workflow `Deploy Jekyll site to Pages`.
3. Открыть последний запуск.
4. Проверить шаг `Build with Jekyll`.
5. Если ошибка связана с YAML/front matter, проверить последние измененные `.md` и `.html` файлы.
6. Если ошибка в Liquid, проверить кавычки, фильтры `relative_url`, `absolute_url`, `jsonify`.

## 10. Минимальная приемка после правок

Правка считается безопасной, если:

- сайт открывается;
- главная страница не ломается;
- меню работает;
- телефон кликается;
- sitemap открывается;
- нет явных ошибок верстки на мобильном;
- нет обещаний гарантированного одобрения ипотеки.
