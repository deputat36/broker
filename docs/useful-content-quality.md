# Качество материалов раздела «Полезно»

## Назначение

Раздел должен помогать человеку разобраться в ситуации и перейти к следующему осмысленному действию. Короткая страница с общими фразами, повторяющимися кнопками и одной ссылкой на услугу не считается полноценным полезным материалом.

Обязательная проверка:

```bash
python3 scripts/audit-useful-content-quality.py _site
```

Она запускается внутри `scripts/post-build-check.sh` для каждого pull request.

## Что проверяется

Для каждого Markdown-файла в `polezno/`, у которого:

- `og_type: article`;
- `permalink` начинается с `/polezno/`;
- permalink не равен корню раздела;

проверяется соответствующая собранная HTML-страница.

Контролируются:

- объём видимого текста внутри `<main>`;
- количество внутренних ссылок без навигационной цепочки;
- количество уникальных тематических маршрутов;
- структура H2/H3;
- ровно один H1;
- наличие CTA на консультацию, онлайн-заявку или телефон;
- наличие собранной страницы для каждого исходника;
- отсутствие дублирующихся permalink.

## Профили

Пороговые значения находятся в:

```text
scripts/useful-content-quality.json
```

Обычная статья должна содержать не менее:

- 270 слов;
- 6 внутренних ссылок;
- 4 уникальных тематических маршрутов;
- 4 заголовков H2/H3.

Навигационный хаб может быть короче, но должен давать более широкую перелинковку. Хабы перечисляются явно в массиве `hubs`.

## Что считается тематической ссылкой

В аудит не засчитываются как самостоятельные тематические маршруты:

- главная;
- корень раздела «Полезно»;
- политика обработки данных;
- согласие на обработку данных;
- ссылка страницы на саму себя;
- assets и 404;
- хлебные крошки.

Ссылка на консультацию или онлайн-заявку подтверждает CTA, но не заменяет содержательные переходы к связанным вопросам.

## Правила изменения порогов

Нельзя снижать порог только ради прохождения одной слабой страницы. Сначала нужно:

1. проверить, не является ли страница хабом;
2. расширить последовательность действий, предупреждения или чек-лист;
3. добавить контекстные ссылки на действительно связанные материалы;
4. убедиться, что текст не дублирует соседнюю страницу;
5. сохранить запрет на меняющиеся ставки и неподтверждённые обещания.

Изменение общих порогов допустимо только после проверки распределения по всему разделу и объяснения причины в pull request.

## Текущая база

На момент введения аудита проверяется 48 материалов. В Pages-артефакте медиана составляет около 411 слов и 12 внутренних ссылок. Отдельный хаб для жителей районов проверяется по профилю `hub`.
