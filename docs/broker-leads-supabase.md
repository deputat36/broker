# Заявки сайта в Supabase

Документ описывает безопасный путь подключения формы заявки на сайте ипотечного брокера.

## Текущее состояние

Сайт на GitHub Pages остается статическим. Прямой интеграции Supabase в коде сайта нет.

В Supabase найден один активный проект:

```text
ofewxuqfjhamgerwzull
```

Этот проект уже используется другими системами, поэтому существующие таблицы и функции `leader`, `nav`, `parket` не трогаем.

## Подготовленная миграция

В репозитории добавлена миграция:

```text
supabase/migrations/202607070001_create_broker_leads.sql
```

Она создает отдельную таблицу:

```text
public.broker_leads
```

Таблица изолирована от существующих модулей и предназначена только для заявок сайта ипотечного брокера.

## Что хранит таблица

- имя клиента;
- телефон;
- город;
- удобное время связи;
- цель ипотеки;
- тип объекта;
- стоимость объекта;
- первоначальный взнос;
- примерный доход;
- наличие маткапитала;
- факт отказа ранее;
- комментарий;
- согласие на обработку персональных данных;
- UTM-метки;
- страницу, с которой пришла заявка.

## Важное правило безопасности

Не вставлять заявки напрямую из браузера в Supabase без антиспама.

Предпочтительный вариант:

```text
форма на сайте -> Edge Function broker-public-lead -> таблица broker_leads
```

Edge Function должна проверять:

- метод запроса;
- CORS-домен сайта;
- обязательный телефон;
- согласие на обработку персональных данных;
- простую антиспам-проверку;
- длину полей.

## Почему anon insert пока выключен

В миграции политика публичной вставки для `anon` оставлена закомментированной.

Это сделано специально: сначала лучше подключить Edge Function и антиспам, а уже потом решать, нужен ли прямой `anon insert`.

## Следующие шаги

1. Применить миграцию в Supabase после подтверждения.
2. Создать Edge Function `broker-public-lead`.
3. Добавить на сайт форму заявки.
4. Протестировать отправку заявки.
5. Добавить уведомления в Telegram, email или CRM.
