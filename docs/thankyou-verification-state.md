# Проверенное состояние страницы благодарности

## Проблема

Страница `/spasibo/` доступна по прямой ссылке. Сам факт её открытия не подтверждает, что форма была отправлена и хотя бы один канал действительно принял обращение.

Кроме того, минимальный localStorage хранит номер последнего обращения до 24 часов. Повторный визит с этим сохранённым ID не должен считаться новой заявкой.

## Три состояния

### Neutral

Страница собирается в нейтральном состоянии:

- заголовок сообщает о проверке подтверждения;
- номер обращения равен `—`;
- передача отмечена как неподтверждённая;
- пользователь может вернуться к форме или связаться с Татьяной напрямую.

### Verified

Текущее успешное состояние включается только при корректном request ID из очищенного fragment-контекста текущего redirect:

- UUID;
- резервный номер формата `IP-YYYYMMDD-XXXXXXXX`.

Только это состояние подтверждает новую отправку и вызывает conversion goal.

### Restored

Если fragment-контекста нет, но в минимальном localStorage сохранился действующий request ID, страница показывает последнее подтверждённое обращение. Текст явно сообщает, что это не новая отправка.

Сценарий, город, телефон, объект и другие данные в localStorage не сохраняются.

## Аналитика

Цель `lead_thankyou_view` отправляется только для текущего redirect с корректным `contextRequestId`.

Повторное открытие того же redirect в рамках сессии дедуплицируется через `sessionStorage`. Request ID используется только в локальном ключе и не передаётся в `dataLayer` или цель аналитики.

Restored-состояние создаёт техническое событие `lead_thankyou_restored_view`, а прямой визит без ID — `lead_thankyou_unverified_view`. Оба события не вызывают conversion goal.

## Автоматическая проверка

```bash
python3 scripts/audit-thankyou-verification-state.py ./_site
```

Аудит требует:

- нейтральное состояние в готовом HTML;
- отсутствие видимого динамического успеха до выполнения JavaScript;
- отдельные `contextRequestId` и `storedRequestId`;
- verified-состояние только для текущего redirect;
- restored-состояние без conversion goal;
- отсутствие conversion goal в неподтверждённой ветке;
- ровно один вызов `trackVerifiedView(contextRequestId)`.

Проверка включена в обязательный post-build suite.
