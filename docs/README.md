# Документация проекта

Этот раздел хранит рабочие технические документы по сайту ипотечного брокера Татьяны Стерликовой.

## Основные документы

- `site-audit-and-autonomous-plan.md` — полный аудит сайта и план автономного развития.
- `live-domain-acceptance.md` — приемка боевого домена, HTTPS, robots, sitemap и поисковых кабинетов.
- `deploy-checklist.md` — проверка GitHub Actions и результата публикации.
- `utm-guidelines.md` — единые правила UTM-разметки для ВКонтакте, объявлений, QR-кодов, печатной рекламы и партнерских размещений.
- `tatyana-photo-brief.md` — техническое задание по подготовке фотографии Татьяны для сайта.
- `photo-implementation-checklist.md` — чек-лист внедрения обработанных фото на сайт.
- `beget-domain-setup.md` — инструкция по подключению домена через Beget и GitHub Pages.
- `broker-leads-supabase.md` — инструкция по будущему подключению заявок сайта к Supabase.

## Что важно соблюдать при развитии сайта

1. Не использовать обещания гарантированного одобрения ипотеки.
2. Не писать, что банк «точно одобрит» заявку.
3. Не обещать конкретную ставку без актуальной проверки условий банка.
4. Уточнять, что финальное решение по заявке, сумме, ставке и объекту принимает банк.
5. Не публиковать отзывы, кейсы и персональные данные клиентов без отдельного согласования.
6. Для клиентов компании «ЭТАЖИ» писать аккуратно: ипотечное сопровождение включено в комиссию компании и отдельно клиентом не оплачивается.

## Автоматические проверки

После сборки workflow запускает два уровня приемки.

### `scripts/post-build-check.sh`

Скрипт проверяет:

- наличие главной, robots, sitemap, CSS, JS и основных изображений;
- корректный `urlset` и достаточное количество URL в sitemap;
- отсутствие 404 в sitemap;
- основной sitemap в robots;
- canonical главной на `sterlikova-ipoteka.ru`;
- отсутствие неподтвержденного обещания `15 мин`;
- отсутствие запрещенных обещаний гарантированного одобрения;
- отсутствие старого технического домена в собранных HTML-файлах.

Наличие `https://deputat36.github.io/broker` в итоговом HTML теперь является ошибкой сборки.

### `scripts/audit-built-site.py`

Python-скрипт проверяет собранные страницы без сторонних библиотек:

- соответствие HTML-страниц и `sitemap.xml`;
- наличие `title`, `meta description`, одного H1 и корректного canonical;
- отсутствие индексируемых страниц вне sitemap;
- корректность всех JSON-LD блоков;
- наличие атрибута `alt` у изображений;
- существование внутренних изображений и файлов;
- битые внутренние ссылки;
- страницы без входящих внутренних ссылок;
- дублирующиеся title и description.

Ошибки останавливают деплой. Возможные страницы-сироты и дубли метаданных пока выводятся как предупреждения для дальнейшей проверки.

## Текущая задача по фотографии

Сейчас на сайте используется `assets/img/tatyana-hero.svg` со встроенным base64-изображением. По ТЗ рекомендуется перейти на отдельные оптимизированные файлы:

- `assets/img/tatyana-hero.webp`;
- `assets/img/tatyana-hero-mobile.webp`;
- `assets/img/tatyana-social.jpg`;
- `assets/img/tatyana-avatar.webp`.

После подготовки файлов нужно заменить вывод изображения в шаблонах на `picture`, чтобы улучшить скорость загрузки и качество адаптива.

10. Отправить sitemap в Яндекс Вебмастер и Google Search Console.
