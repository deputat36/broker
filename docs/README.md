# Документация проекта

Этот раздел хранит рабочие технические документы по сайту ипотечного брокера Татьяны Стерликовой.

## Основные документы

- `tatyana-photo-brief.md` — техническое задание по подготовке фотографии Татьяны для сайта.
- `photo-implementation-checklist.md` — чек-лист внедрения обработанных фото на сайт.
- `beget-domain-setup.md` — инструкция по подключению домена через Beget и GitHub Pages.

## Что важно соблюдать при развитии сайта

1. Не использовать обещания гарантированного одобрения ипотеки.
2. Не писать, что банк “точно одобрит” заявку.
3. Не обещать конкретную ставку без актуальной проверки условий банка.
4. Уточнять, что финальное решение по заявке, сумме, ставке и объекту принимает банк.
5. Не публиковать отзывы, кейсы и персональные данные клиентов без отдельного согласования.
6. Для клиентов компании ЭТАЖИ писать аккуратно: ипотечное сопровождение включено в комиссию компании и отдельно клиентом не оплачивается.

## Текущая задача по фотографии

Сейчас на сайте используется `assets/img/tatyana-hero.svg` со встроенным base64-изображением. По ТЗ рекомендуется перейти на отдельные оптимизированные файлы:

- `assets/img/tatyana-hero.webp`;
- `assets/img/tatyana-hero-mobile.webp`;
- `assets/img/tatyana-social.jpg`;
- `assets/img/tatyana-avatar.webp`.

После подготовки файлов нужно заменить вывод изображения в шаблонах на обычный `picture`, чтобы улучшить скорость загрузки и качество адаптива.

## Следующие рекомендуемые задачи

1. Подготовить и загрузить оптимизированные версии фотографии.
2. Заменить SVG с base64 на WebP/JPG.
3. Проверить главный экран на мобильном экране.
4. Проверить Open Graph-превью в соцсетях и мессенджерах.
5. Подключить Яндекс Метрику и проверить цели.
6. Подключить домен через Beget по инструкции `beget-domain-setup.md`.
7. Отправить sitemap в Яндекс Вебмастер и Google Search Console.
