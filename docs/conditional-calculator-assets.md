# Условная загрузка ипотечного калькулятора

## Проблема

Калькулятор расположен только на `/kalkulyator-ipoteki/`, но его логика раньше находилась в глобальном `assets/js/main.js` и загружалась на всех страницах сайта.

## Решение

- глобальный `main.js` содержит только общую навигацию, аналитику, безопасность ссылок и быстрые контакты;
- расчёт, валидация и мобильные атрибуты полей вынесены в `assets/js/mortgage-calculator.js`;
- отдельный модуль подключается через layout только при `page.url == '/kalkulyator-ipoteki/'`;
- порядок скриптов сохраняется: `main.js` раньше модуля калькулятора, чтобы `window.sendGoal` уже был доступен;
- готовый HTML содержит резервный текст на случай отключённого или не загрузившегося JavaScript;
- успешная инициализация сразу заменяет резервный текст предварительным расчётом.

## Автоматическая проверка

```bash
python3 scripts/audit-conditional-calculator-assets.py ./_site
```

Аудит требует:

- ровно один модуль и одну форму на странице калькулятора;
- отсутствие модуля и формы на остальных HTML-страницах;
- отсутствие калькуляторной логики в глобальном `main.js`;
- наличие расчёта, валидации и аналитической цели в отдельном модуле;
- правильный порядок скриптов;
- понятный резервный текст в готовом HTML.

Проверка включена в обязательный post-build suite.
