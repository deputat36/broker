# Контракт серверного приёма онлайн-заявок

## Текущее состояние

Прямая серверная отправка отключена:

```yaml
lead_capture:
  mode: "disabled"
  endpoint: ""
```

Пока режим отключён, сайт только формирует текст обращения. Клиент самостоятельно передаёт его через SMS, MAX, ВКонтакте, системное меню «Поделиться» или копирование.

Нельзя включать `mode: "direct"`, пока не выполнен весь чек-лист ниже.

## Публичная конфигурация

Файл `_config.yml` находится в открытом репозитории. В нём разрешено хранить только публичный HTTPS URL обработчика и числовые настройки.

Запрещено размещать:

- токен Telegram-бота;
- Supabase `service_role` key;
- пароль CRM;
- API-ключ почтового сервиса;
- любой секрет, позволяющий отправлять сообщения или читать данные.

Секреты должны храниться только на стороне обработчика: Edge Function, сервер, serverless function или защищённая интеграция.

## Включение транспорта

После готовности обработчика:

```yaml
lead_capture:
  mode: "direct"
  endpoint: "https://example.org/api/mortgage-leads"
  timeout_ms: 8000
  min_fill_ms: 3000
```

Клиентский код принимает только HTTPS endpoint. При пустом или некорректном адресе кнопка прямой отправки остаётся скрытой.

## HTTP-запрос

Метод: `POST`

Заголовки:

```text
Accept: application/json
Content-Type: application/json
```

Credentials не передаются. Обработчик не должен требовать cookie или секрет из браузера.

Пример тела запроса:

```json
{
  "schema_version": 1,
  "request_id": "4ad69d4d-e387-4f38-a7ac-c13b87aa160b",
  "form_version": "1",
  "submitted_at": "2026-07-13T12:00:00.000Z",
  "form_fill_ms": 28450,
  "source_page": "/uslugi/semeynaya-ipoteka/",
  "page_url": "https://sterlikova-ipoteka.ru/online-zayavka/?source=...",
  "client": {
    "name": "Анна",
    "phone": "+7 900 000-00-00",
    "city": "Воронеж",
    "preferred_contact": "MAX"
  },
  "mortgage": {
    "scenario": "Семейная ипотека",
    "object_type": "Квартира в новостройке",
    "object_price": "6 000 000 ₽",
    "down_payment": "1 200 000 ₽",
    "income_type": "Официальная работа",
    "bank_history": "Ранее не обращались",
    "comment": "Нужен предварительный разбор"
  },
  "consent": true
}
```

Не следует расширять payload паспортом, СНИЛС, реквизитами карт, кодами подтверждения, фотографиями документов или полным кредитным отчётом.

## Успешный ответ

HTTP status: `200–299`

Рекомендуемый JSON:

```json
{
  "ok": true,
  "lead_id": "lead_20260713_001"
}
```

`lead_id` необязателен, но полезен для поиска заявки и устранения дублей.

## Ошибка

Рекомендуемый формат:

```json
{
  "ok": false,
  "message": "Не удалось сохранить заявку"
}
```

Используйте корректный HTTP status `4xx` или `5xx`. Клиентский код покажет безопасное сообщение и сохранит все резервные способы отправки.

## Обязательные проверки обработчика

Клиентские антиспам-поля не являются полноценной защитой. Сервер обязан самостоятельно:

1. Проверять `Origin` и разрешать только `https://sterlikova-ipoteka.ru`.
2. Настроить CORS только для рабочего домена.
3. Ограничивать частоту запросов по IP, request ID и номеру телефона.
4. Проверять длину и тип каждого поля.
5. Отклонять повторный `request_id`.
6. Нормализовать телефон, не доверяя клиентской разметке.
7. Экранировать данные перед записью в CRM, базу или Telegram.
8. Не возвращать stack trace, токены и внутренние ошибки.
9. Вести минимальный журнал доставки без избыточных персональных данных.
10. Ограничить срок хранения логов и заявок.

Дополнительная CAPTCHA подключается только при подтверждённой необходимости. Её секретный ключ хранится на сервере.

## Идемпотентность

`request_id` создаётся в браузере до формирования заявки. Обработчик должен использовать его как ключ идемпотентности:

- первый запрос сохраняет заявку;
- повтор с тем же `request_id` не создаёт дубль;
- ответ повторного запроса может вернуть прежний `lead_id`.

## Доставка в Telegram, CRM или email

Браузер не должен обращаться к Telegram Bot API, SMTP или CRM с секретом напрямую.

Правильная схема:

```text
Сайт → публичный защищённый endpoint → проверка и rate limit → CRM / Telegram / email
```

## Чек-лист перед активацией

- обработчик развёрнут по HTTPS;
- CORS ограничен рабочим доменом;
- секреты находятся только на сервере;
- rate limit включён;
- request ID используется для защиты от дублей;
- успешный и ошибочный ответы соответствуют контракту;
- заявка действительно приходит выбранному получателю;
- проверены таймаут и недоступность сервера;
- SMS, MAX, ВКонтакте и копирование продолжают работать;
- политика обработки данных обновлена под фактическое серверное хранение;
- определены срок хранения, удаление и доступ к заявкам;
- автоматический аудит обновлён и больше не требует режима `disabled`.

До выполнения всех пунктов endpoint должен оставаться пустым.
