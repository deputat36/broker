# Приватность страницы благодарности

Страница `/spasibo/` показывает клиенту безопасную сводку последней отправленной заявки, но не должна передавать её содержимое в системы аналитики.

## Что допускается показывать клиенту

- технический номер обращения;
- выбранный сценарий консультации;
- город;
- технический статус квалификации.

Эти значения берутся из безопасной локальной сводки `sterlikovaMortgageLastLead`, которая не содержит телефона и полного текста заявки и хранится не более 24 часов.

Для совместимости со старыми ссылками layout временно читает параметры `id`, `scenario` и `status` из URL в объект `window.thankYouContext`, после чего немедленно очищает query string и hash через `history.replaceState`.

Очистка выполняется в `<head>` до возможной инициализации Яндекс Метрики и до запуска `assets/js/main.js`. Поэтому URL страницы, referrer и локальная рекламная атрибуция не должны содержать номер заявки или сведения о её сценарии.

## Что запрещено передавать в аналитику

- `request_id` или отображаемый номер обращения;
- сценарий заявки;
- город;
- квалификацию `hot`, `warm` или `cold`;
- телефон;
- текст заявки;
- банковскую историю и комментарий клиента;
- внутреннее состояние доставки каналов.

Событие страницы благодарности передаётся только в виде:

```js
window.dataLayer.push({ event: 'lead_thankyou_view' });
```

Яндекс Метрика получает только цель `lead_thankyou_view` без параметров.

## Проверка

Автоматический аудит `scripts/audit-thankyou-privacy.py` проверяет:

- наличие ранней очистки URL;
- порядок очистки относительно Метрики;
- отсутствие полей заявки в `dataLayer`;
- валидацию технического номера и allowlist статусов;
- наличие тех же защитных маркеров в собранной странице.

Ручная проверка после публикации:

1. Отправить тестовую заявку.
2. Убедиться, что после перехода отображается чистый адрес `/spasibo/` без query string.
3. Проверить, что номер, сценарий, город и статус видны клиенту из локальной сводки.
4. В режиме Preview Метрики убедиться, что событие `lead_thankyou_view` не содержит параметров заявки.
5. Проверить, что `sterlikovaMortgageTracking` не сохраняет URL с `id`, `scenario` или `status`.
