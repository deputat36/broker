# Атрибуция заявок с главной страницы

## Цель

Главная страница содержит несколько самостоятельных точек входа в онлайн-заявку. Раньше все они вели на одну и ту же форму без явного указания расположения CTA. Теперь переход передаёт контекст, который уже поддерживается действующей системой атрибуции.

## Передаваемые параметры

Каждая ссылка на `/online-zayavka/` с главной передаёт:

- `source=/` — источник обращения: главная страница;
- `scenario` — предзаполняемый сценарий формы;
- `placement` — конкретное расположение CTA на главной.

Внешние рекламные параметры `utm_*`, `gclid`, `yclid`, `vkclid` и другие не подменяются внутренними значениями. Параметр `placement` хранится отдельно в `tracking.current` и попадает в `tracking_json` заявки.

## Список placement

| Placement | Точка входа |
| --- | --- |
| `home_hero_primary` | Основная кнопка первого экрана |
| `home_quick_route` | Карточка быстрого маршрута |
| `home_calc_intro` | CTA рядом с калькулятором на главной |
| `home_intent_online` | Выделенная карточка «Хочу обратиться онлайн» |
| `home_search_cloud` | Ссылка в поисковом облаке |
| `home_complex_inline` | CTA блока сложных случаев |
| `home_trust_panel` | CTA блока доверия |
| `home_services_footer` | CTA после каталога услуг |
| `home_geography_online` | Карточка дистанционной работы |
| `home_prices_intro` | CTA в блоке стоимости |
| `home_useful_footer` | CTA после полезных материалов |
| `home_final_cta` | Финальный CTA страницы |

## Предзаполнение формы

Общие CTA выбирают сценарий «Первичная консультация и подбор ипотеки».

CTA в блоке сложных случаев выбирает «Другая ситуация», чтобы не навязывать пользователю неподходящий тип обращения.

Пользователь может изменить выбранный сценарий вручную. Предзаполнение ускоряет заполнение, но не ограничивает форму.

## Автоматическая проверка

`scripts/audit-homepage-lead-routing.py` проверяет исходник и собранную главную:

- наличие всех placement;
- передачу `source`, `scenario` и `placement` каждой ссылкой;
- корректный сценарий сложного случая;
- отсутствие внутренних `utm_source=site`, `utm_medium=internal` и `lead_source=homepage`;
- поддержку `placement` в allowlist атрибуции;
- сохранение полного `tracking_json` при отправке заявки.
