# Бюджеты производительности Pages

## Назначение

`python3 scripts/audit-performance-budget.py ./_site` проверяет готовый Jekyll-артефакт до публикации. Аудит блокирует незаметный рост HTML, CSS, JavaScript, изображений и локального веса страниц.

## Текущие лимиты

Лимиты хранятся в `scripts/performance-budget.json`.

| Метрика | Лимит |
|---|---:|
| Весь артефакт | 2 900 000 байт |
| Все HTML | 2 650 000 байт |
| Все CSS | 130 000 байт |
| Весь JavaScript | 85 000 байт |
| Все изображения | 50 000 байт |
| Один HTML | 45 000 байт |
| Один CSS | 80 000 байт |
| Один JS | 32 000 байт |
| Одно изображение | 20 000 байт |
| Одна страница с локальными ресурсами | 170 000 байт |
| 95-й процентиль страниц | 115 000 байт |

P95-бюджет снижен со 130 до 115 КБ после условной загрузки ресурсов анкеты. Проверенный результат составляет около 101,5 КБ, поэтому сохранён запас примерно 13% для развития контента.

Лимит одной страницы остаётся выше: `/online-zayavka/` намеренно загружает собственные стили, основной контроллер и вспомогательные модули формы.

## Что уже оптимизировано

- удалён устаревший `tatyana-hero.svg`;
- портреты переведены на WebP;
- `online-application.css` размером 7 090 байт и `online-application.js` размером 27 862 байта подключаются только на странице анкеты;
- на остальных 122 HTML-страницах ресурсы формы отсутствуют;
- одинаковое сравнение артефактов показало уменьшение локального веса обычной страницы на 35 079 байт и снижение P95 примерно на 25,7%.

## Orphan assets

CSS, JavaScript и изображения должны использоваться хотя бы одной HTML-страницей. Единственное разрешённое исключение — `assets/css/style.css`, автоматически создаваемый Jekyll Pages builder.

## Правила изменений

Перед добавлением тяжёлого ресурса проверьте:

1. нужен ли он на каждой странице;
2. можно ли подключать его только на целевом маршруте;
3. можно ли уменьшить его без потери качества;
4. не дублирует ли он существующий файл;
5. не стал ли один маршрут заметно тяжелее остальных.

Бюджет нельзя повышать только ради прохождения разросшейся сборки. Сначала требуется объяснить причину и проверить более лёгкий вариант.

## Границы

Аудит не измеряет Core Web Vitals, задержку сети, CDN-кэш и скорость выполнения JavaScript. Эти параметры проверяются на опубликованном сайте отдельно.
