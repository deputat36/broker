# Бюджеты производительности Pages

## Назначение

`python3 scripts/audit-performance-budget.py ./_site` проверяет фактический Jekyll-артефакт до публикации.

Аудит нужен, чтобы новые статьи, изображения, стили и JavaScript не увеличивали сайт незаметно. Проверка работает без Lighthouse и внешних сервисов, поэтому результат воспроизводим в каждом pull request.

## Что измеряется

Проверяются:

- общий несжатый вес Pages-артефакта;
- суммарный вес HTML, CSS, JavaScript и изображений;
- максимальный размер одного HTML, CSS, JS или изображения;
- локальный вес каждой страницы: HTML плюс реально подключённые CSS, JS, favicon, preload и изображения;
- 95-й процентиль локального веса страниц;
- неподключённые asset-файлы.

Open Graph-изображение считается используемым asset, но не прибавляется к обычной загрузке страницы: его получает робот социальной сети, а не браузер посетителя.

## Текущие лимиты

Лимиты хранятся в `scripts/performance-budget.json`.

| Метрика | Лимит |
|---|---:|
| Весь артефакт | 2 900 000 байт |
| Все HTML | 2 650 000 байт |
| Все CSS | 130 000 байт |
| Весь JavaScript | 85 000 байт |
| Все изображения | 50 000 байт |
| Один HTML | 45 000 байт |
| Один CSS | 80 000 байт |
| Один JS | 32 000 байт |
| Одно изображение | 20 000 байт |
| Одна страница с локальными ресурсами | 170 000 байт |
| 95-й процентиль страниц | 130 000 байт |

Лимиты установлены с небольшим запасом относительно проверенного Pages-артефакта. Их нельзя повышать только для того, чтобы пропустить случайно разросшийся файл.

## Orphan assets

Аудит требует, чтобы CSS, JavaScript и изображения из `assets/` использовались хотя бы одной собранной HTML-страницей.

Единственное разрешённое исключение:

- `assets/css/style.css` — автоматически создаваемый Jekyll Pages theme-asset. Сайт его не подключает, но текущий Pages builder добавляет файл в артефакт.

Любое новое исключение должно иметь отдельное техническое объяснение. Нельзя добавлять файл в allowlist только потому, что его забыли подключить или удалить.

## Выполненная очистка

Устаревший `assets/img/tatyana-hero.svg` удалён. Клиентские страницы после обязательного prebuild используют оптимизированные WebP-версии:

- `tatyana-hero.webp`;
- `tatyana-hero-mobile.webp`;
- `tatyana-avatar.webp`.

Удаление уменьшает артефакт примерно на 15,6 КБ и исключает возврат старого встроенного JPEG-контейнера.

## Как менять сайт

Перед добавлением тяжёлого ресурса нужно сначала проверить:

1. нужен ли он на каждой странице;
2. можно ли подключать его только на целевом маршруте;
3. можно ли уменьшить размер без потери читаемости;
4. не дублирует ли он существующий CSS, JS или изображение;
5. не стал ли один материал заметно тяжелее остальных.

Следующий приоритет оптимизации — условная загрузка ресурсов онлайн-заявки только на странице анкеты. До безопасного изменения общего layout текущий вес зафиксирован бюджетом и не должен расти.

## Границы

Этот аудит не измеряет Core Web Vitals, задержку сети, кэширование CDN, производительность устройства или выполнение JavaScript в браузере. Эти параметры проверяются отдельно на опубликованном сайте.
