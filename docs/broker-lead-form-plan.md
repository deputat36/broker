# План формы заявки через Supabase

Документ описывает возможную будущую интеграцию формы заявки на сайте ипотечного брокера.

## Зачем нужна форма

Сейчас основные действия сайта:

- звонок;
- MAX через копирование номера;
- ВКонтакте;
- переход на консультацию.

Форма может понадобиться позже, если нужно собирать обращения прямо с сайта.

## Минимальные поля

Для первой версии достаточно:

- имя;
- телефон;
- город;
- тип объекта;
- примерная стоимость;
- первоначальный взнос;
- комментарий;
- согласие на обработку персональных данных;
- страница, с которой отправлена заявка.

## Рекомендуемая архитектура

1. Отдельная таблица заявок в Supabase.
2. Отдельная Edge Function, например `broker-public-lead`.
3. Проверка обязательных полей на стороне функции.
4. Ограничение частоты отправки.
5. Сохранение технических данных: дата, страница, user agent.
6. Уведомление ответственному человеку после успешной заявки.

## Что не использовать

- Не вставлять service role key на сайт.
- Не отправлять заявку напрямую в таблицу без серверной проверки.
- Не хранить лишние персональные данные.
- Не смешивать заявки ипотечного сайта с заявками других проектов без явной структуры.

## Перед запуском

До реализации нужно согласовать:

- точный список полей;
- текст согласия;
- куда отправлять уведомления;
- нужно ли хранить источник рекламы;
- нужна ли интеграция с CRM;
- отдельный проект Supabase или использование существующего.
