# Сайт ипотечного брокера Татьяны Стерликовой

SEO-сайт для сбора обращений по ипотечным консультациям и сопровождению в Борисоглебске, Борисоглебском городском округе, Грибановском и Поворинском районах.

Рабочий репозиторий: `deputat36/broker`.

Основной публичный адрес:

`https://sterlikova-ipoteka.ru/`

Технический адрес GitHub Pages после подключения домена остается запасным и не должен использоваться в рекламе.

## Главная задача сайта

Сайт должен приводить клиентов из поиска и рекламы, объяснять услуги простым языком и переводить посетителя в обращение:

- звонок по телефону `8 903 025-08-07`;
- копирование телефона для MAX;
- переход на страницу ВКонтакте `https://vk.com/tatyanasterlikova`;
- переход на страницу бесплатной консультации;
- предварительный расчет платежа в ипотечном калькуляторе.

Публичная форма заявок пока не включена на сайте. Backend для будущих заявок подготовлен в Supabase, но форму нужно включать только после отдельного тестирования обработчика, согласия на обработку данных и антиспама.

## Supabase-заявки

В Supabase подготовлен backend для будущей формы:

- таблица `public.broker_leads`;
- RLS включен;
- публичная вставка через `anon` не включалась;
- Edge Function `broker-public-lead` активна;
- исходник функции хранится в `supabase/functions/broker-public-lead/index.ts`;
- миграция хранится в `supabase/migrations/202607070001_create_broker_leads.sql`.

Публичную форму на сайте включать только после тестирования отправки, CORS, ошибок, согласия на обработку персональных данных и антиспам-поля.

## Публикация на GitHub Pages

Рекомендуемый вариант:

1. Открыть `Settings → Pages`.
2. В качестве source выбрать `GitHub Actions`.
3. Запустить workflow `Deploy Jekyll site to Pages` вручную или дождаться запуска после push в `main`.

Запасной вариант:

1. `Settings → Pages`.
2. Source: `Deploy from a branch`.
3. Branch: `main`.
4. Folder: `/root`.

## Работа с доменом

Основной домен проекта:

`sterlikova-ipoteka.ru`

В репозитории уже сделано:

1. Добавлен файл `CNAME` со строкой `sterlikova-ipoteka.ru`.
2. В `_config.yml` задано:
   - `url: "https://sterlikova-ipoteka.ru"`;
   - `baseurl: ""`.
3. Обновлен `site.webmanifest` под корень домена.
4. `robots.txt`, `sitemap.xml`, canonical и Open Graph используют `absolute_url` и подтянут новый домен при сборке.

Что нужно проверить после настройки DNS:

1. В `Settings → Pages` указан custom domain `sterlikova-ipoteka.ru`.
2. DNS-записи домена указывают на GitHub Pages.
3. GitHub Pages показывает успешную проверку DNS.
4. Включен `Enforce HTTPS`.
5. Открываются адреса:
   - `https://sterlikova-ipoteka.ru/`;
   - `https://sterlikova-ipoteka.ru/sitemap.xml`;
   - `https://sterlikova-ipoteka.ru/robots.txt`.
6. После запуска домен добавлен в Яндекс Вебмастер и Google Search Console.

## Яндекс Метрика

Код Метрики подготовлен, но не выводится, пока в `_config.yml` пустое значение:

`yandex_metrika_id: ""`

После создания счетчика нужно вставить ID:

`yandex_metrika_id: "12345678"`

После этого сайт начнет отправлять цели:

- `phone_click` — клик по телефону.
- `vk_click` — переход во ВКонтакте.
- `max_copy` — копирование телефона для MAX.
- `consultation_click` — переход на страницу консультации.
- `contacts_click` — переход на страницу контактов.
- `calculator_input` — первое изменение данных в ипотечном калькуляторе.
