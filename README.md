# Сайт ипотечного брокера Татьяны Стерликовой

SEO- и рекламный сайт для сбора обращений по ипотечным консультациям и сопровождению.

Репозиторий: `deputat36/broker`  
Публичный адрес: `https://sterlikova-ipoteka.ru/`

Технический адрес GitHub Pages остаётся запасным и не используется в рекламе.

## Позиционирование

Сайт объединяет два направления:

1. Частный ипотечный брокер Татьяна Стерликова.
2. Ипотечное сопровождение клиентов компании «ЭТАЖИ» в рамках сделки через компанию.

Первичную онлайн-консультацию можно запросить из любого города России.

Отдельные локальные SEO-разделы подготовлены для:

- Борисоглебска и Борисоглебского городского округа;
- Грибановского района;
- Поворино и Поворинского района.

Локальные страницы не ограничивают дистанционные обращения. Возможность дальнейшего сопровождения зависит от региона, банка, объекта и согласованной задачи.

## Как принимаются заявки

Приём реализован по той же схеме, что в проекте `deputat36/bm` — «Новостройки Борисоглебска».

Рабочий канал:

- Web3Forms принимает данные формы;
- email-копия отправляется получателю, настроенному в Web3Forms;
- после успешного технического ответа пользователь переходит на `/spasibo/`;
- на странице благодарности показываются номер обращения, сценарий, город и технический приоритет;
- телефон и полный текст заявки в localStorage не сохраняются.

Все публичные настройки канала находятся в `_config.yml`:

```yaml
lead_capture:
  mode: "web3forms"
  web3forms_access_key: "UUID"
  web3forms_endpoint: "https://api.web3forms.com/submit"
  endpoint: ""
  thank_you_path: "/spasibo/"
  timeout_ms: 8000
  min_fill_ms: 3000
```

Web3Forms access key является публичным идентификатором формы и определяет получателя писем. При смене рабочего email заменяется только `lead_capture.web3forms_access_key`; JavaScript редактировать не нужно.

Дополнительный Supabase или CRM endpoint пока пустой. В будущем режим можно заменить на `hybrid`, сохранив Web3Forms как независимую email-копию.

Фактический получатель текущего ключа подтверждается отдельным smoke-тестом в issue №8.

## Данные онлайн-заявки

Форма собирает:

- имя;
- телефон;
- город или населённый пункт;
- предпочтительный способ связи;
- ипотечный сценарий;
- тип и примерную стоимость объекта;
- первоначальный взнос;
- способ подтверждения дохода;
- историю заявок, одобрений и отказов;
- комментарий;
- страницу-источник;
- первое и последнее рекламное касание;
- UTM-метки и рекламные click-id;
- технический request ID;
- время заполнения;
- согласие на обработку данных.

Форма не предназначена для паспорта, СНИЛС, банковских реквизитов, кодов подтверждения, фотографий документов или полного кредитного отчёта.

## Атрибуция и предзаполнение

`assets/js/main.js` сохраняет:

- первое касание;
- последнее касание;
- текущие UTM-метки;
- `gclid`, `yclid`, `ymclid`, `vkclid` и другие рекламные идентификаторы;
- URL, заголовок страницы и referrer.

Срок хранения рекламной атрибуции в браузере — 90 дней с последнего посещения. Устаревшая запись удаляется при следующем открытии сайта. Для ручной очистки доступна функция `window.clearSiteTrackingData()` и стандартная очистка данных сайта в браузере.

Страницы услуг и региональные посадочные страницы передают параметр `source`. Форма автоматически подставляет сценарий, объект и регион, когда это возможно.

После успешной отправки сохраняется только безопасная сводка без телефона и полного текста заявки. Она получает срок 24 часа и затем удаляется при следующем открытии `/spasibo/`.

## Квалификация

Заявка получает технический статус:

- `hot`;
- `warm`;
- `cold`.

Статус используется только для приоритета обработки. Он не означает вероятность банковского одобрения.

## Защита формы

Клиентская часть содержит:

- honeypot;
- минимальное время заполнения;
- request ID;
- защиту от повторного клика после успешной отправки;
- валидацию UUID-формата Web3Forms access key;
- разрешение только HTTPS-адресов доставки;
- таймаут через AbortController;
- `credentials: omit`;
- безопасный fallback;
- отдельные success, error и timeout-состояния.

При ошибке Web3Forms пользователь сохраняет готовый текст и может отправить его через SMS, MAX, ВКонтакте, Web Share API или скопировать вручную.

Workflow `.github/workflows/pages.yml` выполняет:

1. `node --check` для JavaScript.
2. Компиляцию Python-скриптов.
3. Сборку Jekyll.
4. Общий аудит HTML, ссылок и Schema.org.
5. Аудит CTA и конверсионных страниц.
6. SEO-аудит 18 услуг.
7. Аудит региональных страниц и смысловой уникальности.
8. Аудит Web3Forms-конфигурации, UUID-формата ключа, email-доставки, согласия, `/spasibo/`, сроков хранения, атрибуции и fallback.
9. Проверку, что Web3Forms key не зашит прямо в JavaScript.
10. Загрузку Pages artifact только после успешных проверок.
