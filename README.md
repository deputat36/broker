# Сайт ипотечного брокера Татьяны Стерликовой

SEO- и рекламный сайт для сбора обращений по ипотечным консультациям и сопровождению.

Рабочий репозиторий: `deputat36/broker`.

Основной публичный адрес:

`https://sterlikova-ipoteka.ru/`

Технический адрес GitHub Pages после подключения домена остаётся запасным и не должен использоваться в рекламе.

## Позиционирование и география

Сайт объединяет два направления:

1. Частный ипотечный брокер Татьяна Стерликова.
2. Ипотечное сопровождение клиентов компании «ЭТАЖИ» в рамках сделки через компанию.

Первичную онлайн-консультацию можно запросить из любого города России.

Отдельные локальные SEO-разделы подготовлены для:

- Борисоглебска и Борисоглебского городского округа;
- Грибановского района;
- Поворино и Поворинского района.

Локальные страницы не ограничивают дистанционные обращения из других населённых пунктов. Возможность дальнейшего сопровождения зависит от региона, банка, объекта и согласованной задачи.

## Основная конверсия

Сайт переводит посетителя в обращение через:

- онлайн-заявку `/online-zayavka/`;
- звонок по телефону `8 903 025-08-07`;
- копирование телефона для MAX;
- переход во ВКонтакте `https://vk.com/tatyanasterlikova`;
- SMS;
- системное меню «Поделиться»;
- бесплатную первичную консультацию;
- предварительный расчёт в ипотечном калькуляторе.

## Как работает онлайн-заявка сейчас

Форма:

- доступна клиентам из любого города;
- собирает основные вводные по ипотечной ситуации;
- сохраняет страницу-источник;
- предзаполняет сценарий, тип объекта и локальный регион;
- создаёт request ID;
- формирует структурированный текст;
- предлагает SMS, MAX, ВКонтакте, копирование и Web Share API;
- не требует передачи паспорта или документов;
- не сохраняет сведения на сервере сайта автоматически.

Прямой серверный endpoint подготовлен на уровне клиентской архитектуры, но отключён:

```yaml
lead_capture:
  mode: "disabled"
  endpoint: ""
  timeout_ms: 8000
  min_fill_ms: 3000
```

При пустом endpoint кнопка прямой доставки скрыта. Резервные способы связи работают независимо от будущего backend.

## Защита онлайн-заявки

В клиентском коде подготовлены:

- honeypot;
- минимальное время заполнения;
- request ID;
- защита от повторного клика после успешной доставки;
- HTTPS-only endpoint;
- таймаут через AbortController;
- `credentials: omit`;
- безопасный fallback при ошибке сервера;
- отдельные состояния success, error и timeout.

Клиентские меры не заменяют серверный rate limit, проверку Origin, CORS, валидацию и идемпотентность.

Контракт будущего endpoint:

`docs/lead-endpoint-contract.md`

## Состояние Supabase backend

В репозитории присутствуют legacy-исходники:

- `supabase/functions/broker-public-lead/index.ts`;
- `supabase/migrations/202607070001_create_broker_leads.sql`.

Они не подключены к текущей форме и не считаются готовым production backend.

Подтверждённые блокеры:

- Edge Function ожидает старый плоский payload;
- текущая форма формирует вложенный контракт `schema_version: 1`;
- в таблице нет отдельного `request_id` и уникального индекса;
- нет подтверждённого серверного rate limit;
- не подтверждено уведомление Татьяне;
- живой деплой функции и миграции через GitHub не проверен;
- политика обработки данных пока описывает режим без серверного хранения.

Полный аудит:

`docs/supabase-lead-backend-audit.md`

До устранения блокеров `lead_capture.mode` должен оставаться `disabled`.

## Яндекс Метрика

Код Метрики не выводится, пока в `_config.yml` пустое значение:

```yaml
yandex_metrika_id: ""
```

Карта целей:

`docs/analytics-events.md`

Основные группы событий:

- телефон, ВКонтакте и MAX;
- калькулятор;
- переход к форме;
- определение источника;
- начало заполнения;
- ошибка валидации;
- подготовка заявки;
- SMS, Share, VK и копирование;
- будущая прямая доставка: attempt, success, error и timeout.

`online_application_prepare` не означает подтверждённую доставку заявки.

## Публикация на GitHub Pages

1. Открыть `Settings → Pages`.
2. Выбрать source `GitHub Actions`.
3. Указать custom domain `sterlikova-ipoteka.ru`.
4. Проверить DNS.
5. Включить `Enforce HTTPS`.
6. Проверить сайт, `sitemap.xml` и `robots.txt`.
7. Добавить домен в Яндекс Вебмастер и Google Search Console.
